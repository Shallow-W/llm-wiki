# 任务卸载（Task Offloading）

## 定义

将计算任务从资源受限的边缘设备（或边缘节点）转移到资源更充足的计算节点（边缘服务器或云）执行的过程。核心权衡是在**通信开销**（传输数据）和**计算收益**（更快处理）之间取得平衡。

## 关键属性

- **卸载决策**：二值决策（本地执行 vs. 卸载）+ 目标节点选择
- **部分卸载**：将任务分割为多个子任务，分别在不同节点执行
- **二元性**：传输数据越多 → 通信延迟越高；但目标节点算力越强 → 处理延迟越低
- **关联决策**：卸载到哪里 → 取决于哪里部署了所需模型 → 取决于模型放置策略

## 与部署/路由的关系

```
任务卸载 → 决定任务去哪个节点 → 该节点必须有对应的服务/模型
                                      ↑
                               这是"部署"问题
任务到达节点后 → 交给哪个实例处理 → 这是"路由"问题
```

任务卸载、服务部署、请求路由三者形成闭环决策链。
