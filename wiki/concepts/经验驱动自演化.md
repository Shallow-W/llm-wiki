# 经验驱动自演化（Experience-Driven Self-Evolution）

## 定义

一种 LLM Agent 学习范式，通过"在线交互 → 离线自蒸馏 → 策略演化"的闭环循环，让 Agent 从自身交互经验中持续提炼可复用的策略原则，并通过强化学习将这些原则内化为策略参数，实现无需外部教师模型的持续自我改进。

## 核心思想

与现有的三种 Agent 学习范式形成对比：

| 范式 | 机制 | 缺陷 |
|------|------|------|
| **无状态执行** | 每次交互独立，不保留经验 | 操作性失忆，无法积累知识 |
| **原始轨迹学习** | 检索并模仿过去的完整轨迹 | 难以抽象和泛化，仅表面模仿 |
| **外部教师蒸馏** | 依赖更强 LLM 提炼经验 | 认知不对齐，Agent 自身策略未变 |
| **经验驱动自演化** | 自蒸馏 + RL 内化，完整闭环 | 蒸馏质量受限于基座模型能力 |

直觉：就像学生做数学题——做完题后反思"为什么做对/做错了"（自蒸馏），提炼出解题策略（原则），下次遇到类似问题时先回忆策略再解题（在线检索），不断练习使策略成为本能（RL 内化）。

## 关键属性

### 闭环生命周期

1. **在线交互**：Agent 在 Think-Act-Observe 循环中解决任务，可随时检索经验库中的策略原则引导推理，生成高质量轨迹
2. **离线自蒸馏**：冻结策略参数，从交互轨迹中提炼结构化的策略原则（成功→引导原则，失败→警示原则），并维护经验库质量
3. **策略演化**：通过 GRPO 等强化学习算法，利用经验引导下收集的轨迹更新策略参数，学会有效利用蒸馏智慧

### 经验库管理三机制

- **去重**：一级同题去重 + 二级跨题语义等价检查（embedding 相似度 + LLM 判断）
- **动态评分**：$s(p) = \frac{c_{\text{succ}}(p) + 1}{c_{\text{use}}(p) + 2}$（Laplace 平滑的经验效用分数）
- **剪枝**：定期删除低于阈值的低质量原则

### 认知对齐效应

关键发现：当基座模型足够大（~3B 参数）时，由 Agent 自身策略蒸馏的原则比外部更强教师模型蒸馏的原则更有效。原因：自蒸馏产生的原则与 Agent 自身的推理模式和失败模式更"对齐"。

## 与其他概念的关系

### 与 Agent RL 的关系

经验驱动自演化是 [[Agent RL]] 的一个具体应用场景。它将 RL 用于 Agent 策略的持续演化，而非常规的单任务优化。关键区别：RL 的训练数据是经验原则引导下的轨迹，学到的策略是"如何有效利用自身智慧"而非通用推理。

### 与记忆机制的关系

经验驱动自演化可视为一种**文本级外部记忆 + RL 内化**的方案：

| 方案 | 记忆载体 | 读写方式 | 与 Agent RL 的关系 |
|------|---------|---------|-------------------|
| [[δ-mem（在线关联记忆）]] | 参数级（矩阵状态） | delta-rule 在线更新 | 无关，纯前向计算 |
| [[Titans]] | 架构级（深度 MLP） | 训练时学习 | 无关，测试时记忆 |
| **经验驱动自演化** | 文本级（原则库） | 自蒸馏 + 检索 | RL 用于策略内化 |

### 与 RAG 的区别

RAG 检索的是**外部事实性知识**，解决的是"不知道"的问题。经验驱动自演化检索的是**自身策略性智慧**，解决的是"不会做"的问题。两者互补而非替代。

## 局限

1. **能力天花板**：蒸馏质量受限于基座模型能力，小模型（<1.5B）的自蒸馏效果显著弱于教师蒸馏
2. **经验内化困难**：直接将检索到的原则通过梯度下降内化到参数中会引入噪声，当前方案仍然依赖"检索式"而非"内化式"
3. **任务泛化待验证**：目前仅在 QA 任务上验证，具身交互、创意生成等场景的适用性未知
