# 自蒸馏（Agent 策略自蒸馏）

## 定义

在 Agent 语境下，自蒸馏是指 Agent 利用**自身的策略模型**（而非外部更强的教师模型）分析过去的交互轨迹，从中提炼出抽象的、可复用的策略原则的过程。核心区别于传统知识蒸馏：这里没有教师-学生对齐的范式，而是 Agent 扮演"自我反思者"的角色。

## 核心机制

### 蒸馏流程

1. **轨迹审查**：Agent 的策略模型 $\pi_\theta$（冻结参数）审查每条历史交互轨迹
2. **专家角色扮演**：通过精心设计的 prompt 让模型以"专家"视角分析轨迹
3. **原则生成**：根据轨迹成败分别生成
   - **成功轨迹** → 引导原则（Guiding Principle）：描述"什么策略有效"
   - **失败轨迹** → 警示原则（Cautionary Principle）：描述"什么错误应避免"
4. **结构化表示**：每条原则 = 自然语言描述 + $(subject, predicate, object)$ 三元组列表

### 认知对齐（Cognitive Alignment）

自蒸馏最关键的特性是**认知对齐**——蒸馏出的原则与 Agent 自身的推理方式天然匹配。

实验证据（来自 EvolveR）：

| 模型规模 | 自蒸馏平均 | GPT-4o-mini 教师蒸馏平均 | 胜者 |
|---------|-----------|------------------------|------|
| 0.5B | 0.150 | 0.220 | 教师 |
| 1.5B | 0.270 | 0.290 | 教师 |
| 3B | **0.382** | 0.370 | **自蒸馏** |

在 3B 规模，自蒸馏反超教师蒸馏。原因分析：
- 自蒸馏的原则反映了 Agent **自身的失败模式和推理习惯**，针对性更强
- 外部教师模型（即使是 GPT-4o-mini）生成的原则是"旁观者视角"，可能包含 Agent 无法执行的策略
- 但小模型（<1.5B）自身分析能力不足，蒸馏出低质量原则，此时教师模型仍有优势

### 与传统知识蒸馏的区别

| 维度 | 传统知识蒸馏 | Agent 策略自蒸馏 |
|------|------------|----------------|
| **目标** | 模型压缩（大→小） | 策略知识提炼（经验→原则） |
| **教师** | 更大的预训练模型 | Agent 自身（冻结策略） |
| **知识类型** | 输出分布 / 特征表示 | 自然语言策略原则 |
| **传递方式** | 软标签 / 特征匹配 | Prompt 引导的文本生成 |
| **对齐问题** | 模型容量差异导致信息损失 | 认知不对齐（教师-学生视角不同） |
| **应用时机** | 训练时 | 离线反思阶段 |

## 关键属性

- **无需外部教师**：消除了对更强 LLM 的依赖，尤其在 3B+ 规模
- **策略抽象**：从具体轨迹到通用原则，跨越"表面模仿"到"深层理解"
- **双面学习**：既从成功中提取"该做什么"，也从失败中提取"不该做什么"
- **尺度依赖**：效果随基座模型能力增长，存在一个"自蒸馏超越教师蒸馏"的临界规模
