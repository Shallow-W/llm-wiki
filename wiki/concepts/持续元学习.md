# 持续元学习（Continual Meta-Learning）

## 定义

一种将**元学习**（学会快速适配新任务）与**持续学习**（在任务序列中不断积累知识而不遗忘）相结合的学习范式。目标是让学习系统在非平稳任务流中持续进化，不仅在每个任务上表现更好，而且逐步提高其自身的适配能力——"学会更好地学习"。

## 核心思想

### 两种时间尺度的统一

持续元学习的核心洞察是：**快速适配和慢速优化是天然互补的**，可以通过统一的元模型联合维护：

| 时间尺度 | 机制 | 速度 | 目标 |
|---------|------|------|------|
| **快适配** | 梯度无关的行为调整（如技能注入） | 秒级 | 从单次失败中即时提取可复用知识 |
| **慢优化** | 梯度驱动的策略更新（如 RL） | 分钟-小时级 | 跨多轨迹改善底层策略能力 |

两种机制形成**良性循环（virtuous cycle）**：更好的策略产生更有信息量的失败供快适配蒸馏，更丰富的快适配知识产出更高奖励的轨迹供慢优化利用。

### 元模型结构

在 MetaClaw 的具体实例中，元模型定义为：

$$\mathcal{M} = (\theta, \mathcal{S})$$

其中 $\theta$ 是基座策略参数（通过 RL 更新），$\mathcal{S}$ 是技能库（通过无梯度演化更新）。给定任务 $\tau$：

$$a \sim \pi_\theta(\cdot \mid \tau, \text{Retrieve}(\mathcal{S}, \tau))$$

### 与经典元学习的关系

经典元学习（如 MAML）的内外循环结构可映射到持续元学习：

| MAML | 持续元学习（MetaClaw 实例） |
|------|--------------------------|
| 内循环：在支撑集上快速适配 | 技能驱动快适配：从失败中合成新技能 |
| 外循环：在查询集上更新元参数 | 机会主义策略优化：用后适配轨迹更新 $\theta$ |
| 离线、回合制 | 在线、异步、流式 |
| 适配是梯度计算 | 适配是自然语言合成（离散空间） |

## 关键属性

### 1. 支撑-查询分离（Support-Query Separation）

持续元学习的关键数据治理原则：用于驱动快适配的数据（支撑集 $D^{\text{sup}}$）与用于评估后适配行为的数据（查询集 $D^{\text{qry}}$）必须严格分离。混用会导致策略优化基于过时奖励信号更新，违反元学习目标。

在在线、异步设置中，这需要显式的版本化机制（如 [[技能代际版本化（Skill Generation Versioning）]]）来强制执行。

### 2. 非平稳任务流

与经典元学习假设固定离线任务分布不同，持续元学习面对的是非平稳分布 $p_t(\tau)$：任务类型、难度和频率随时间漂移。系统必须同时跟踪分布变化并适配。

### 3. 不间断服务约束

生产部署中的持续元学习必须在不中断服务的前提下进行。这要求：
- 快适配机制**零停机**（通过 prompt 注入而非参数更新）
- 慢优化机制**延迟到空闲窗口**（通过调度器管理）

### 4. 双重知识积累

- **离散知识积累**：技能库 $\mathcal{S}$ 单调递增，每个技能代际 $\mathcal{S}_{g+1} \supseteq \mathcal{S}_g$ 代表增长的操作知识
- **连续知识积累**：策略参数 $\theta$ 通过梯度更新逐步改善底层能力

## 与相关范式的区别

| 范式 | 目标 | 在线？ | 适配能力提升？ | 权重优化？ |
|------|------|--------|-------------|-----------|
| **经典元学习** (MAML) | 学会快速适配 | 否（离线） | 是 | 是 |
| **持续学习** (EWC, GEM) | 顺序学任务不遗忘 | 是 | 否（不改善适配能力） | 是 |
| **在线元学习** (Finn et al. 2019) | 在线元学习 | 是 | 是（简单网络） | 是 |
| **经验驱动自演化** (EvolveR) | Agent 自我改进 | 部分（离线蒸馏+在线交互） | 是 | 是 |
| **持续元学习** (MetaClaw) | 生产级持续进化 | 是（完全在线） | 是 | 是（机会主义） |

## 局限

1. **空闲窗口依赖**：策略优化依赖用户空闲时段，高频使用场景下可能长期无法更新权重
2. **技能膨胀风险**：技能库单调递增不加剪枝，长期运行可能面临检索效率和知识冗余问题
3. **评估有限**：当前仅在模拟基准（MetaClaw-Bench）和单一跨域任务上验证
4. **遗忘问题未充分讨论**：策略参数的持续更新可能导致早期技能相关的知识被覆盖
