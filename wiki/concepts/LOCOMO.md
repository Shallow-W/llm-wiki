# LOCOMO（长对话记忆基准）

## 定义

LOCOMO（Long-term Conversational Memory）是评估 LLM Agent 长对话记忆能力的标准基准数据集。它包含 10 段扩展的多会话对话，每段约 600 轮、26,000 tokens，并配有约 200 个带标准答案的问题，覆盖单跳、多跳、时序和开放域四种推理类型。

## 关键属性

### 数据集规模

| 指标 | 数值 |
|------|------|
| 对话段数 | 10 |
| 每段平均轮数 | ~600 |
| 每段平均 token 数 | ~26,000 |
| 每段平均问题数 | ~200 |
| 总问题数 | ~2,000 |

### 问题类型

| 类型 | 说明 | 难度 |
|------|------|------|
| **单跳（Single-Hop）** | 答案位于单个对话轮次中 | 低 |
| **多跳（Multi-Hop）** | 需综合多个会话中的分散信息 | 高 |
| **时序（Temporal）** | 需推理事件顺序、时间关系 | 中-高 |
| **开放域（Open-Domain）** | 需结合对话内容推断外部知识 | 中 |
| ~~对抗性（Adversarial）~~ | ~~测试识别不可答问题的能力~~ | ~~-~~ |

> 对抗性问题因缺乏标准答案，在大多数评估中被排除。

### 对话特征

- 两段对话之间通常有"间隔事件"（intervening life events），模拟真实世界中对话不连续的场景
- 参与者讨论日常生活或过去事件
- 关键信息可能分散在数百轮对话中

## 评估指标

### 性能指标

| 指标 | 说明 | 局限 |
|------|------|------|
| **F1 Score** | 词汇重叠 | 对语义错误不敏感（如"3月"vs"7月"仍得高分） |
| **BLEU-1** | 一元词匹配 | 同上 |
| **LLM-as-a-Judge (J)** | GPT-4o 评判事实准确性 | 更贴近人类判断，但有随机性 |

### 部署指标

- **Token Consumption**：检索上下文 token 数
- **Search Latency**：检索时间
- **Total Latency**：检索 + 生成总时间

## 在 LOCOMO 上的代表性结果

### 当前最优（截至 2025-04）

| 方法 | Overall J | 特点 |
|------|-----------|------|
| Full-context | 72.90 | 准确率最高但延迟 17s，不可扩展 |
| **Mem0g** | **68.44** | 图记忆，最佳平衡 |
| **Mem0** | **66.88** | 文本记忆，最低延迟（1.44s） |
| Zep | 65.99 | 商业平台，开放域最强 |
| OpenAI Memory | 52.90 | 专有方案，时序表现差 |

### 关键发现

1. **全上下文准确率最高但不可行**：72.90 J，但 p95 延迟 17 秒
2. **记忆系统显著优于 RAG**：最佳 RAG 仅 60.97 J，比 Mem0 低 6 分
3. **文本记忆 vs 图记忆各有优势**：Mem0 单跳/多跳更强，Mem0g 时序/开放域更强
4. **延迟与质量的权衡**：Mem0 以 1.44s 延迟达到 66.88 J，性价比最优

## 局限

- **领域单一**：仅覆盖日常对话，缺乏医疗、法律、教育等专业领域
- **语言单一**：主要为英语
- **对话轮次有限**：600 轮虽长，但真实应用可能涉及数千轮
- **评估指标不完美**：F1/BLEU 对语义错误不敏感，LLM-as-a-Judge 有随机性
