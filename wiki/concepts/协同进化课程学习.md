# 协同进化课程学习（Co-Evolutionary Curriculum Learning）

## 定义

一种让两个功能互补的 Agent（课程 Agent 与执行 Agent）从同一基座模型初始化，通过共生竞争关系协同进化的学习范式。课程 Agent 负责生成自适应难度的训练任务，执行 Agent 负责学习解决这些任务；两者能力相互驱动，形成正反馈循环。

## 核心思想

传统课程学习（Curriculum Learning）依赖人工设计或由固定启发式规则控制难度 progression。协同进化课程学习则将"课程设计"本身作为一个学习问题，让课程生成器（Generator）与课程消费者（Solver）共同进化：

```
传统课程学习：
人工/固定规则 → 预定义课程序列 → 模型学习

协同进化课程学习：
课程 Agent π_θ ←── 奖励信号 ──→ 执行 Agent π_φ
    ↓ 生成任务                    ↓ 解决任务
    └──────── 能力反馈 ←──────────┘
```

**关键洞察**：课程 Agent 的奖励直接来自执行 Agent 的表现——任务越"恰到好处地挑战"执行 Agent，课程 Agent 获得的奖励越高。这创造了一个自动调节的难度 progression 机制。

## 架构组件

### 课程 Agent（Curriculum Agent）

- **角色**：任务生成器
- **目标**：生成使执行 Agent 不确定性最大化的任务
- **训练信号**：执行 Agent 的自洽度 $\hat{p}(x)$（越接近 0.5 奖励越高）
- **策略优化**：GRPO 或类似 RL 算法

### 执行 Agent（Executor Agent）

- **角色**：任务求解器
- **目标**：成功解决课程 Agent 生成的任务
- **训练信号**：任务解决正确率（通过多数投票伪标签）
- **策略优化**：ADPO 或标准 RL 算法

### 共生竞争关系

| 维度 | 课程 Agent 的视角 | 执行 Agent 的视角 |
|------|------------------|------------------|
| **竞争** | 要让执行 Agent "头疼" | 要证明自己的能力 |
| **共生** | 执行 Agent 越强，可生成的任务越复杂 | 课程 Agent 越好，训练数据质量越高 |
| **结果** | 任务复杂度螺旋上升 | 解决能力同步提升 |

## 与相关范式的对比

| 范式 | 课程来源 | 进化机制 | 能力上限 | 代表工作 |
|------|---------|---------|---------|---------|
| **人工课程** | 人类专家设计 | 无 | 人类知识边界 | Bengio et al. (2009) |
| **自挑战** | 模型自身生成 | 单 Agent | 模型固有知识 | R-Zero, Absolute Zero |
| **协同进化课程** | 专用 Agent 生成 | 双 Agent 竞争 | 工具扩展边界 | **Agent0** |
| **对抗生成** | 判别器反馈 | GAN 式 | 训练稳定性 | 无直接对应 |

## 关键设计决策

### 1. 奖励设计：不确定性即信号

课程 Agent 的核心奖励是执行 Agent 的**不确定性**（以自洽度 $\hat{p}$ 为代理），而非任务本身的客观难度。这确保了：
- 任务始终位于执行 Agent 的"最近发展区"
- 避免生成过难（执行 Agent 完全无法解决）或过易（已掌握）的任务

### 2. 多样性维持

为防止课程 Agent 陷入局部最优（反复生成相似任务），需要显式的多样性机制：
- **重复惩罚**：基于 BLEU 相似度的聚类惩罚
- **工具使用奖励**：鼓励生成需要不同工具组合的任务
- **格式约束**：确保输出符合预期结构

### 3. 工具作为能力杠杆

将外部工具（如代码解释器）集成到执行 Agent 中，不仅增强其问题解决能力，更重要的是：
- 扩展了可生成任务的空间（可以包含需要数值计算、符号操作的问题）
- 为课程 Agent 提供了额外的奖励维度（工具使用频率）
- 创造了"工具使用 → 能力增强 → 更复杂任务 → 更多工具使用"的良性循环

## 在 Agent0 中的实现

Agent0 的协同进化循环（3 次迭代）：

| 指标 | Iter 1 | Iter 2 | Iter 3 | 趋势 |
|------|--------|--------|--------|------|
| 执行 Agent 数学 AVG | 55.1 | 56.5 | 58.2 | 持续上升 |
| 固定执行 Agent 对新任务通过率 | 64.0% | 58.5% | 51.0% | 任务难度上升 |
| 平均工具调用次数 | 1.65 | 2.10 | 2.60 | 工具依赖加深 |

## 优点

1. **自适应难度**：无需人工干预，难度自动匹配当前能力水平
2. **无限扩展**：不受人类知识边界限制，可生成超越人类已知复杂度的问题
3. **正反馈循环**：两个 Agent 相互驱动，形成持续改进的螺旋
4. **数据效率**：每个生成的任务都位于学习最有效的前沿区域

## 局限

1. **训练不稳定**：双 Agent 同时训练增加了优化复杂度，需要仔细平衡学习率
2. **模式崩溃风险**：课程 Agent 可能收敛到特定类型的问题（需要多样性机制对抗）
3. **评估成本**：每个候选任务需要执行 Agent 多次采样评估，计算开销大
4. **能力不对齐**：课程 Agent 可能过度优化特定维度（如工具使用）而忽略其他能力
5. **冷启动问题**：初始阶段两个 Agent 都较弱，生成的任务质量可能较低
