# 技能代际版本化（Skill Generation Versioning）

## 定义

一种在在线持续元学习系统中强制执行**支撑-查询分离**（support-query separation）的数据治理机制。通过为每条收集的轨迹标记其对应的技能库版本（代际），确保策略优化始终基于"技能适配后"的行为数据，防止过时奖励信号污染梯度更新。

## 核心问题

在 Agent 持续学习系统中，技能库 $\mathcal{S}$ 会随着经验积累而演化（$\mathcal{S}_g \to \mathcal{S}_{g+1}$）。一条在 $\mathcal{S}_g$ 下收集的轨迹 $(\tau_i, \xi_i)$，其奖励 $r_i$ 反映的是**旧技能上下文**中的表现。如果该轨迹被直接用于 RL 训练：

1. 策略优化会因一个**已被新技能纠正的失败**而惩罚参数 $\theta$
2. 优化目标从"后适配表现"偏移为"前适配表现"
3. 违反了元学习目标函数

$$\theta_{t+1} = \theta_t + \alpha \nabla_\theta \mathbb{E}_{(\tau, \xi, g') \sim \mathcal{B}} \left[ \mathcal{R}(\pi_\theta(\cdot \mid \tau, \mathcal{S}_{g'})) \right]$$

此公式要求缓冲区 $\mathcal{B}$ 中每条轨迹的奖励反映其对应代际 $\mathcal{S}_{g'}$ 下的真实表现。

## 机制

### 数据分类

每条轨迹被标记其收集时的技能代际 $g_i$：

| 数据类型 | 符号 | 含义 | 去向 |
|---------|------|------|------|
| **支撑数据** | $D_g^{\text{sup}}$ | 在 $\mathcal{S}_g$ 下收集的失败轨迹，触发 $\mathcal{S}_g \to \mathcal{S}_{g+1}$ | 被技能演化器消费，**排除出 RL 缓冲区** |
| **查询数据** | $D_{g+1}^{\text{qry}}$ | 在 $\mathcal{S}_{g+1}$ 生效后收集的轨迹，反映后适配行为 | **仅此类进入 RL 缓冲区** |

### 版本化协议

1. **标记**：每条收集的样本 $(\tau_i, \xi_i, r_i)$ 标记当前技能代际 $g_i$
2. **分离**：揭示失败的轨迹进入支撑集 $D_g^{\text{sup}}$；成功的轨迹标记版本后加入 RL 缓冲区 $\mathcal{B}$
3. **刷新**：当技能代际从 $g$ 推进到 $g+1$ 时，清空 $\mathcal{B}$ 中所有版本 $\leq g$ 的样本
4. **门控**：策略优化仅在 $\mathcal{B}$ 积累了足够查询数据后启动

### 具体示例

```
时间线：
  t1: 技能库 = S_0，收集轨迹 A（失败）→ 进入 D_sup_0
  t2: 技能库 = S_0，收集轨迹 B（成功）→ 标记 g=0，进入 B
  t3: |D_sup_0| ≥ 阈值 → 演化：S_0 → S_1
      → 清空 B 中 g ≤ 0 的样本（轨迹 B 被移除）
  t4: 技能库 = S_1，收集轨迹 C（成功）→ 标记 g=1，进入 B
  t5: 技能库 = S_1，收集轨迹 D（成功）→ 标记 g=1，进入 B
  t6: 用户空闲 & |B| ≥ batch_size → 用 {C, D} 进行 RL 更新
```

## 关键属性

- **在线异步安全**：在任务顺序到达、技能演化异步触发的在线环境中有效运作
- **零信息泄漏**：严格保证 RL 训练数据不包含前适配行为的奖励信号
- **代际混合训练**：RL 缓冲区可包含多个代际的查询数据（$g' \leq g^*$），每条轨迹用其对应代际 $\mathcal{S}_{g'}$ 评估
- **数据门控**：样本不足时自动推迟训练，避免高方差梯度估计

## 与相关概念的对比

| 机制 | 解决的问题 | 适用场景 |
|------|-----------|---------|
| **经验回放** (Experience Replay) | 打破时间相关性 | 持续学习/离线 RL |
| **课程学习** (Curriculum Learning) | 控制训练数据难度 | 训练加速 |
| **技能代际版本化** | 防止过时奖励污染 | 在线持续元学习中的数据有效性 |

## 局限

1. **数据浪费**：每次技能演化都清空旧查询数据，如果演化频繁，可能导致 RL 缓冲区长期处于饥饿状态
2. **仅适用于离散适配事件**：假设技能库的变化是离散的"代际"跳跃，对连续渐变的适配方式不直接适用
3. **不解决策略遗忘**：版本化保护了数据有效性，但不防止策略参数更新对早期技能知识的覆盖
