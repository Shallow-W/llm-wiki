# 零数据自举（Zero-Data Bootstrapping）

## 定义

一种让 LLM Agent 在**完全不依赖人工标注数据**的条件下，通过自我生成训练信号和自我对抗/协同进化来提升能力的范式。Agent 从自身的交互和推理过程中提取学习信号，实现"无中生有"的能力增长。

## 核心思想

传统 LLM 训练 pipeline：

```
预训练 → SFT（人工标注）→ RLHF/RLVR（人工偏好/验证）
              ↑ 需要大量人工数据
```

零数据自举 pipeline：

```
基座模型 → 自生成任务 → 自评估/自验证 → 自训练 → 能力增强
    ↑___________________________________________|
                    闭环循环
```

**关键洞察**：LLM 本身具有足够的先验知识来生成合理的任务和评估标准，关键在于设计机制让这种自我生成过程产生 progressively harder 的训练信号。

## 主要技术路线

### 路线一：单 Agent 自挑战（Self-Challenging）

同一模型既生成任务又解决任务，通过自我一致性或验证规则评估。

| 方法 | 核心机制 | 能力上限 | 代表工作 |
|------|---------|---------|---------|
| R-Zero | 自生成问题 + 自一致性验证 | 模型固有知识 | Huang et al. (2025) |
| Absolute Zero | 自生成问题 + 代码执行器验证 | 模型知识 + 验证能力 | Zhao et al. (2025) |
| SPIRAL | 零和博弈自对弈 | 模型固有知识 | Liu et al. (2025a) |

**共同局限**：能力天花板受限于模型固有知识，容易陷入课程停滞（curriculum stagnation）。

### 路线二：双 Agent 协同进化（Co-Evolution）

两个功能不同的 Agent 从同一基座初始化，通过竞争/合作关系共同进化。

| 方法 | 角色分工 | 关键创新 | 代表工作 |
|------|---------|---------|---------|
| Socratic-Zero | 提问 Agent + 回答 Agent | 依赖外部 API | Wang et al. (2025d) |
| **Agent0** | 课程 Agent + 执行 Agent | **工具集成 + 自适应课程** | Xia et al. (2025) |

**关键突破**：工具集成打破了模型固有知识的天花板。

### 路线三：经验驱动自演化（Experience-Driven）

从交互轨迹中提炼策略原则，通过 RL 内化。

| 方法 | 核心机制 | 记忆形式 | 代表工作 |
|------|---------|---------|---------|
| EvolveR | 在线交互 → 离线自蒸馏 → 策略演化 | 文本级原则库 | Wu et al. (2025b) |

## 能力天花板问题

零数据自举面临的核心挑战：**模型无法生成超出自身能力范围的任务**。

```
模型能力边界
    ┌─────────────────────┐
    │  可生成的任务空间    │
    │  （受限于模型知识）  │
    │                     │
    │  ┌───────────────┐  │
    │  │ 可解决的任务   │  │
    │  │ （更小）       │  │
    │  └───────────────┘  │
    └─────────────────────┘
```

**解决方案对比**：

| 方案 | 机制 | 效果 | 代价 |
|------|------|------|------|
| 外部工具 | 引入代码解释器等 | 扩展可解决任务空间 | 需要工具基础设施 |
| 多轮交互 | 增加上下文复杂度 | 提升任务有效难度 | 增加推理成本 |
| 对抗生成 | 两个模型相互挑战 | 挖掘模型知识边界 | 训练不稳定 |
| 外部模型 | 调用更强模型辅助 | 直接获取更高能力 | 依赖外部 API |

Agent0 的核心创新是**工具 + 双 Agent + 多轮**的三重组合，同时从多个维度突破天花板。

## 自监督信号来源

零数据自举需要某种形式的"自监督"来替代人工标签：

| 信号类型 | 具体形式 | 适用场景 | 可靠性 |
|---------|---------|---------|--------|
| **自一致性** | 多次采样答案的多数投票 | 有确定答案的问题 | 中（受模型偏见影响） |
| **可验证规则** | 代码执行、数学验证 | 代码、数学问题 | 高（客观正确性） |
| **对抗评估** | 另一 Agent 的解决成功率 | 开放式任务 | 中（取决于评估 Agent 能力） |
| **环境反馈** | 沙箱执行结果 | 工具使用任务 | 高（客观结果） |

Agent0 综合使用了自一致性（伪标签）和环境反馈（工具执行结果）两种信号。

## 与相关概念的关系

### 与 Self-Supervised Learning 的区别

传统自监督学习（如 BERT 的 MLM、GPT 的 next-token prediction）利用数据本身的结构生成监督信号，但**仍然需要大量原始数据**。零数据自举更进一步——**不需要任何外部数据**，完全从模型自身的生成和交互中学习。

### 与 RLVR 的区别

RLVR（Reinforcement Learning from Verifiable Rewards）使用可验证奖励（如代码执行结果），但通常**仍需要人工策划的问题集**。零数据自举连问题集都是自我生成的。

### 与蒸馏的区别

知识蒸馏需要教师模型（通常是更强模型）提供软标签。零数据自举**不依赖外部教师**，完全自力更生。

## 优点

1. **彻底摆脱数据瓶颈**：无需人工标注，可无限扩展
2. **突破人类知识边界**：可能发现人类未想到的问题和解题策略
3. **成本效益**：省去数据收集和标注的高昂成本
4. **持续进化**：可无限迭代，理论上能力没有上限（受计算资源限制）

## 局限与挑战

1. **课程停滞**：单 Agent 方法容易陷入能力天花板
2. **伪标签噪声**：自一致性并非完美标签，尤其在困难任务上
3. **计算成本**：需要大量采样和评估，计算开销巨大
4. **安全与对齐**：自主进化可能产生未预期的行为，需要价值约束
5. **评估困难**：缺乏标准答案时难以客观衡量进步
6. **领域限制**：目前主要在数学推理和代码上验证，泛化性待验证
