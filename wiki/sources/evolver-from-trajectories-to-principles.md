# EvolveR: Self-Evolving LLM Agents Through an Experience-Driven Lifecycle

## 一句话总结

EvolveR 提出了一种闭环的经验驱动生命周期框架，让 LLM Agent 能够通过"在线交互 → 离线自蒸馏 → 策略演化"的循环，将原始交互轨迹提炼为可复用的策略原则，并通过 GRPO 强化学习内化这些原则，实现 Agent 的持续自我改进。

## 研究背景与动机

当前 LLM Agent 在工具使用方面表现出色，但存在一个根本性缺陷：**操作性失忆（operational amnesia）**——每次交互都被视为独立事件，Agent 无法从过去的成功或失败中系统性地学习。

已有方法尝试解决这一问题，但各有不足：

| 范式 | 代表方法 | 核心缺陷 |
|------|---------|---------|
| 无状态执行 | ReAct | 每次交互后丢弃经验，Agent 内在策略不变 |
| 原始轨迹学习 | ExpeL | 检索原始轨迹，难以抽象和泛化，仅模仿而非提炼 |
| 外部教师蒸馏 | Reflexion + 外部 LLM | 依赖更强的教师模型，Agent 自身策略仍未改变；且存在"认知不对齐"问题 |

EvolveR 的核心洞察是：**Agent 需要一个完整的闭环生命周期**——不仅要能从经验中提炼智慧，还要能将这种智慧内化为自身策略的一部分，形成"越用越强"的正向循环。

## 核心方法

### 总体架构

EvolveR 由三个相互连接的核心组件构成，形成闭环：

```
在线交互（Online Interaction）
    ↓ 生成高质量轨迹
策略演化（Policy Evolution via GRPO）
    ↓ 更新策略参数
离线自蒸馏（Offline Self-Distillation）
    ↓ 提炼策略原则 → 喂入在线交互
    ↑_________________________|
```

### 符号表

| 符号 | 含义 | 维度/说明 |

| $a_t \in \mathcal{A}$ | 动作 | 三种动作之一 |
| $\mathcal{E}$ | 经验库（Experience Base） | 存储已蒸馏的策略原则集合 |
| $p$ | 原则（Principle） | 自然语言描述 + 结构化三元组 |
| $P_k$ | 检索到的原则集合 | Top-$k$ 条相关原则 |
| $\tau$ | 轨迹（Trajectory） | 一次完整交互的记录 |
| $c_{\text{succ}}(p)$ | 原则 $p$ 的成功次数 | 历史使用中引导成功的累计 |
| $c_{\text{use}}(p)$ | 原则 $p$ 的使用次数 | 历史被检索使用的累计 |
| $s(p)$ | 原则 $p$ 的度量分数 | 经验效用评估 |

| $\theta_{\text{prune}}$ | 剪枝阈值 | 默认 0.3 |
| $R(\tau)$ | 轨迹奖励 | 复合奖励函数 |
| $w_o, w_f$ | 奖励权重 | 结果奖励 / 格式奖励权重 |
| $G$ | GRPO 组大小 | 每个提示的采样轨迹数（默认 8） |

### 动作空间设计

Agent 的动作空间 $\mathcal{A}$ 包含三种动作：

1. **`<search experience>`**：查询内部经验库 $\mathcal{E}$，检索相关策略原则。环境返回检索到的原则作为观测。
2. **`<search knowledge>`**：查询外部知识库（搜索引擎），获取事实信息。环境返回检索到的信息作为观测。
3. **`<answer>`**：输出最终答案，结束交互。

关键区别：`<search experience>` 和 `<search knowledge>` 是两种本质不同的检索——前者检索的是 Agent 自己从过去经验中提炼的**策略性智慧**，后者检索的是外部**事实性信息**。

### 第一阶段：离线经验自蒸馏（Offline Self-Distillation）

此阶段 Agent 的**策略模型参数** $\theta$ 冻结。$\theta$ 是基座 LLM（如 Qwen2.5-3B）的全部可学习参数——即整个 Transformer 的权重矩阵。冻结意味着这些权重不参与梯度更新，模型"停工思考"——只做推理（生成原则、做语义判断），不做训练。此阶段的目标是从原始轨迹中提炼结构化知识（原则库 $\mathcal{E}$），而非改变模型本身。

#### 原则蒸馏

利用 Agent 自身的策略模型 $\pi_\theta$ 分析历史交互轨迹。通过精心设计的 prompt 让模型扮演"专家"角色，审查每条轨迹并提炼核心策略洞察：

- **成功轨迹** → 引导原则（Guiding Principle）：描述成功策略
- **失败轨迹** → 警示原则（Cautionary Principle）：描述应避免的陷阱

每条原则包含两部分：
1. **自然语言描述**：一句话概括核心建议
2. **结构化三元组**：$(subject, predicate, object)$ 格式的知识表示

直觉：这不是简单的"记录做了什么"，而是提炼"为什么这么做有效/无效"的抽象洞察。

#### 去重与整合

新原则不是简单添加，而是经过严格的两级整合流程：

**第一级：同题去重**

由于 GRPO 对同一问题采样多条轨迹（$G=8$），同一问题会产生多条相似原则。使用 $\pi_\theta$ 进行成对语义等价检查，每个语义等价簇仅保留一条代表。

**第二级：跨题整合**

对每条唯一原则，执行两阶段匹配：
1. 通过 embedding 相似度从 $\mathcal{E}$ 检索最相似的已有原则
2. 用 $\pi_\theta$ 做二元语义等价判断

更新规则：

$$\mathcal{E} \leftarrow \begin{cases} \mathcal{E} \cup \{p_{\text{cand}}\} & \text{if } \max_{p \in \mathcal{E}} \text{sim}(p_{\text{cand}}, p) < \theta_{\text{sim}} \\ \text{Merge}(\mathcal{E}, p^*, \tau_{\text{src}}) & \text{otherwise} \end{cases}$$

其中 $p^* = \arg\max_{p \in \mathcal{E}} \text{sim}(p_{\text{cand}}, p)$，$\text{sim}(\cdot, \cdot)$ 为余弦相似度，$\theta_{\text{sim}} = 0.85$。

直觉：新知识要么作为独立条目加入，要么作为已有原则的新证据被合并——确保经验库是"精炼"的而非"膨胀"的。

#### 质量控制：动态评分

每条原则跟踪使用次数和成功次数，计算经验效用分数：

$$s(p) = \frac{c_{\text{succ}}(p) + 1}{c_{\text{use}}(p) + 2}$$

分子 +1、分母 +2 是 Laplace 平滑，防止早期样本量不足时的极端值。

定期剪枝：删除 $s(p) < \theta_{\text{prune}}$（默认 0.3）的原则。

### 第二阶段：在线交互（Online Interaction）

Agent 在 Think-Act-Observe 循环中工作，核心创新在于**检索到的原则如何改变 Agent 行为**：

- Agent 随时可发出 `<search experience>` 动作
- 检索到的原则 $P_k$ 提供的不是事实信息，而是**启发式引导**，塑造 Agent 的后续推理
- 例如，检索到"对于比较类问题，在得出结论前应收集双方数据"，Agent 的内部推理（`<thinking>`）和后续 `<search knowledge>` 动作都会被这一原则引导

**双重视角理解在线阶段**：
1. 当下层面：利用原则高效解决当前任务
2. 未来层面：原则引导下的高质量轨迹为下一轮离线蒸馏提供优质素材

### 第三阶段：策略演化（Policy Evolution via RL）

前两个阶段中，离线蒸馏产生了原则库 $\mathcal{E}$，在线交互在原则引导下收集了轨迹 $\mathcal{D}$。但这些经验仍然停留在"外部提示"层面——模型每次需要通过 `<search experience>` 主动检索原则才能利用它们。**第三阶段的目标是将这种外部引导内化为模型自身的策略能力**，使模型"本能地"知道如何推理，而不再需要每次显式检索。

#### 被更新的参数是什么？

被更新的参数 $\theta$ 是**基座 LLM 的权重**——与第一阶段冻结的是同一组参数。具体实现上，EvolveR 使用 **LoRA** 对 Qwen2.5 的注意力权重进行低秩适配（通过 LLaMA Factory 框架），而非全量微调：

- **可训练参数**：LoRA 的低秩矩阵 $A$ 和 $B$（秩未明确指定，但参数量远小于全量微调）
- **冻结参数**：Qwen2.5 的全部预训练权重
- **RL 学习率**：$1 \times 10^{-6}$（非常小，确保稳定）
- **效果**：只有 LoRA 参数被 GRPO 更新，预训练知识被完整保留

> **为什么不是全量微调？** 全量微调 175B 模型需要 1.2TB VRAM。即使用 LoRA，GRPO 的训练仍需 8×A100 (80GB) 跑约 39.4 小时（3B 模型）。全量微调的代价和过拟合风险使得 LoRA + GRPO 成为更实际的选择。

#### 复合奖励函数

GRPO 优化需要一个标量奖励信号来评估每条轨迹的质量。EvolveR 设计了一个复合奖励函数：

$$R(\tau) = w_o R_{\text{outcome}}(\tau) + w_f R_{\text{format}}(\tau)$$

参数设置：$w_o = 1.0$，$w_f = 0.1$，即**结果奖励权重远高于格式奖励**——最终目标是答对问题，格式只是辅助塑形信号。

**结果奖励** $R_{\text{outcome}}$（稀疏、二元）：

$$R_{\text{outcome}}(\tau) = \text{EM}(a_{\text{pred}}, a_{\text{gold}})$$

- $a_{\text{pred}}$：从轨迹 $\tau$ 中提取的模型最终答案
- $a_{\text{gold}}$：标准答案
- EM：精确匹配（exact match），答对 = 1，答错 = 0
- 这是**稀疏奖励**：只在轨迹结束时给出，中间步骤没有信号

**格式奖励** $R_{\text{format}}$（密集、塑形）：

$$R_{\text{format}}(\tau) = \frac{\mathbb{I}(\tau_{\text{complete}}) \cdot R_{\text{think}}(\tau) + R_{\text{search}}(\tau)}{2}$$

格式奖励解决"稀疏奖励导致 RL 信号不足"的问题——即使最终答案错了，只要推理过程结构合理，仍能得到正向反馈。

各组成部分：

| 组成 | 含义 | 设计直觉 |
|------|------|---------|
| $\mathbb{I}(\tau_{\text{complete}})$ | 轨迹完整性指示器 | 必须包含 `<thinking>`、至少一种搜索、`<answer>`，否则整个 think 分数为 0 |
| $R_{\text{think}}(\tau)$ | 推理步数评分 | 鼓励约 6 步推理；过多反而扣分（防止啰嗦和重复） |
| $R_{\text{search}}(\tau)$ | 搜索评分 = 多样性 + 数量 | 多样性：同时用 `<search experience>` 和 `<search knowledge>` 得 0.5，仅一种得 0.2；数量：每次额外搜索 +0.1，上限 0.5 |

> **为什么需要格式奖励？** 纯结果奖励（答对/答错）太稀疏——大多数轨迹都是"答错"的，模型收到的几乎全是零梯度信号。格式奖励提供了一个密集的塑形信号，引导模型学会"正确的推理格式"（先思考→搜索经验→搜索知识→再思考→回答），即使答案不对也能获得中间奖励。

#### GRPO 算法详解

**Group Relative Policy Optimization（GRPO）** 是 DeepSeek 提出的一种在线 RL 算法，是 PPO 的简化变体。它的核心创新是**用组内平均奖励替代学习型价值函数**，避免训练一个单独的 critic 模型。

##### 第一步：组采样（Group Sampling）

对于每个输入问题 $q$，从当前策略 $\pi_\theta$ 中采样 $G$ 条轨迹：

$$\{\tau_1, \tau_2, \ldots, \tau_G\} \sim \pi_\theta(\cdot | q)$$

EvolveR 设置 $G = 8$（每步采样 128 prompts，每个 prompt 8 条轨迹）。

##### 第二步：计算组相对优势（Group Relative Advantage）

这是 GRPO 的核心——**不需要训练价值函数**，而是用同一组内其他轨迹的奖励作为基线：

$$\hat{A}_i = \frac{R(\tau_i) - \text{mean}(\{R(\tau_j)\}_{j=1}^G)}{\text{std}(\{R(\tau_j)\}_{j=1}^G)}$$

- 分子：第 $i$ 条轨迹的奖励减去组内平均奖励
- 分母：组内奖励的标准差（归一化）
- **直觉**：一条轨迹好不好，不是看绝对奖励值，而是看它相对于同一问题下其他轨迹的表现。比平均好 → 正优势（鼓励），比平均差 → 负优势（抑制）

> **为什么这比 PPO 更适合 Agent RL？** PPO 需要训练一个独立的 value function $V(s)$ 来估计每个状态的基线——这要求额外的模型参数和训练开销。GRPO 直接用同一问题的多条轨迹互为基线，零额外参数，且天然适应不同难度的问题（简单问题奖励普遍高，困难问题奖励普遍低，组内相对消除了这种偏差）。

##### 第三步：重要性采样比率（Importance Sampling Ratio）

$$\rho_t(\theta) = \frac{\pi_\theta(a_t | h_t)}{\pi_{\text{old}}(a_t | h_t)}$$

- $\pi_\theta$：当前策略（正在更新的模型）
- $\pi_{\text{old}}$：采样轨迹时使用的旧策略
- $\rho_t$ 衡量"在当前策略下产生这个动作的概率"与"在旧策略下产生这个动作的概率"的比值
- $\rho_t > 1$：当前策略比旧策略更倾向于这个动作
- $\rho_t < 1$：当前策略比旧策略更不倾向于这个动作

##### 第四步：裁剪目标（Clipped Objective）

$$J_{\text{GRPO}}(\theta) = \mathbb{E}_{\tau \in \mathcal{D}} \left[ \sum_{t=1}^{|\tau|} \min\left( \rho_t(\theta) \hat{A}_t,\  \text{clip}(\rho_t(\theta), 1 - \epsilon, 1 + \epsilon) \hat{A}_t \right) - \beta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}] \right]$$

逐项拆解：

| 项 | 公式 | 作用 |
|----|------|------|
| **未裁剪项** | $\rho_t(\theta) \hat{A}_t$ | 标准策略梯度：如果优势为正，增大该动作概率；如果为负，减小 |
| **裁剪项** | $\text{clip}(\rho_t, 1-\epsilon, 1+\epsilon) \hat{A}_t$ | 限制 $\rho_t$ 在 $[1-\epsilon, 1+\epsilon]$ 范围内（$\epsilon = 0.2$），防止单步更新过大导致策略崩溃 |
| **取最小值** | $\min(\text{未裁剪}, \text{裁剪})$ | 悲观估计：在有利方向上取较小值（保守前进），在不利方向上取较大值（允许回退） |
| **KL 惩罚** | $-\beta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$ | 防止更新后的策略偏离参考策略太远（$\beta = 0.001$，$\pi_{\text{ref}}$ 是 RL 开始前的模型） |

> **裁剪的直觉**：假设旧策略以 0.3 的概率选择了动作 A，当前策略以 0.9 的概率选择。$\rho_t = 0.9/0.3 = 3.0$。如果 $\epsilon = 0.2$，裁剪后 $\rho_t = 1.2$。这意味着"策略在这个动作上变强了，但我们不信任太大的变化，所以把置信度限制在 1.2 倍"。这是一种**信任区域**约束。

##### GRPO 训练的超参数

| 参数 | 值 | 含义 |
|------|-----|------|
| 每步采样 | 128 prompts | 每个 RL 步处理 128 个不同问题 |
| 组大小 $G$ | 8 | 每个问题采样 8 条轨迹 |
| RL 学习率 | $1 \times 10^{-6}$ | LoRA 参数的学习率（非常小） |
| 裁剪 $\epsilon$ | 0.2 | 重要性采样比率的裁剪范围 |
| KL 系数 $\beta$ | 0.001 | KL 散度惩罚的权重 |

#### "关闭循环"的关键洞察

GRPO 的优化对象 $\mathcal{D}$ 不是随机收集的轨迹，而是在**经验原则引导下**收集的轨迹。这意味着：

1. **输入端**：Agent 在线交互时，检索到的原则 $P_k$ 塑造了 Agent 的推理和搜索行为
2. **输出端**：这些被原则引导的高质量轨迹被 GRPO 用来更新策略参数 $\theta$
3. **闭环**：更新后的 $\theta$ 让模型"内化"了原则的智慧——即使下次不检索原则，模型也已经学会了更好的推理模式

> **没有 RL 的 EvolveR 会怎样？** 原则只是外部提示。每次推理都需要 `<search experience>` 动作去检索，增加了推理开销和错误可能。GRPO 将这种"每次都要查笔记才能答好题"的模式转化为"已经把笔记内容记在脑子里了"的内在能力。这才是真正的"关闭循环"。

#### 完整生命周期的参数状态

| 阶段 | 模型参数 $\theta$ | 经验库 $\mathcal{E}$ | 活动描述 |
|------|-------------------|----------------------|---------|
| **离线蒸馏** | 🔒 冻结 | ✏️ 更新 | 模型分析轨迹，提炼/去重/剪枝原则 |
| **在线交互** | 🔒 冻结 | 📖 只读 | 模型在原则引导下与环境交互，收集轨迹 |
| **策略演化 (RL)** | ✏️ 更新（LoRA） | 🔒 冻结 | GRPO 用收集的轨迹更新模型参数 |

这个三阶段循环不断重复：蒸馏→交互→演化→蒸馏→...，每次循环都让经验库更丰富、策略模型更强。

### 冷启动（Cold Start）

在 RL 训练前，用约 700 条 NQ + HotpotQA 训练集样本，由 GPT-4o 生成格式正确的 CoT 交互轨迹，通过 SFT 教会模型基本的交互格式。LoRA 微调，3 epochs，学习率 $1 \times 10^{-4}$。

## 实验结果

### 主实验（Table 1）

在 7 个 QA benchmark 上评估，包括域内（NQ、HotpotQA）和域外（TriviaQA、PopQA、2Wiki-MultiHopQA、Musique、Bamboogle）。

**Qwen2.5-3B 结果：**

## 消融实验/分析

### 1. 自蒸馏 vs 教师蒸馏（Table 2）

| 模型 | 自蒸馏平均 | GPT-4o-mini 教师蒸馏平均 | 结论 |
|------|-----------|------------------------|------|
| 0.5B | 0.150 | **0.220** | 小模型自蒸馏能力不足，教师更有效 |
| 1.5B | 0.270 | **0.290** | 差距缩小 |
| 3B | **0.382** | 0.370 | **反转！自蒸馏超越教师蒸馏** |

核心洞察：**认知对齐（Cognitive Alignment）**——当模型足够大时，由自身策略蒸馏出的原则与自身推理方式更"对齐"，比外部更强模型的蒸馏更有效。

### 2. 经验检索的必要性（Table 3）

| 模型 | 有检索 | 无检索 | 差距 |
|------|--------|--------|------|
| 0.5B | 0.150 | 0.078 | -48% |
| 1.5B | 0.270 | 0.123 | -54% |
| 3B | 0.382 | 0.340 | -11% |

所有规模上移除检索都导致显著下降，证明经验检索是不可或缺的。

### 3. 组件归因（Table 4）

| 原则蒸馏 | 原则检索 | RL | 平均分 |
|---------|---------|-----|--------|
| 否 | 否 | 否 | 0.134 |
| 否 | 否 | 是 | 0.325 |
| 是 | 是 | 否 | 0.357 |
| 是 | 否 | 是 | 0.340 |
| 是 | 是 | 是 | **0.382** |

关键发现：
- RL 单独提供基线提升（0.134 → 0.325）
- 经验检索与 RL 是互补的：检索提供推理引导，RL 优化利用检索原则的策略
- 三者协同效果最强

### 4. SFT vs RL 训练对比（Table 8）

| 方法 | 平均分 |
|------|--------|
| EvolveR (SFT) | 0.357 |
| EvolveR (RL/GRPO) | **0.382** (+7%) |

SFT 只学习表面动作模式，GRPO 从成功和失败轨迹中学习"什么该做、什么该避免"。

### 5. 原则质量分析（Table 6）

人工标注 100 条原则：

| 分数层级 | 理想（正确且可操作） | 模糊（正确但不可操作） | 错误/误导 |
|---------|---------------------|----------------------|----------|
| 低分段 (0.30-0.44) | 26% | 50% | 24% |
| 高分段 (0.44-0.75) | **82%** | 10% | 8% |

动态评分机制有效过滤了低质量原则，高分段中可操作原则占比从 26% 提升至 82%。

### 6. 经验库扩展性（Table 7）

| 原则数 | 检索延迟 | 平均分 |
|--------|---------|--------|
| 4,213 | 0.04s | 0.301 |
| 13,435 | 0.06s | 0.372 |
| 23,448 | 0.12s | 0.382 |
| 32,805 | 0.16s | 0.396 |
| 45,083 | 0.19s | **0.410** |
| 49,934 | 0.20s | 0.387 |

经验库扩展至近 5 万条原则时，检索延迟仅 0.20s，性能保持稳定甚至在 45k 时达到峰值。

### 7. 经验内化尝试（Table 11）

尝试将检索到的原则直接"吸收"进模型参数（不解码器 loss mask），结果轻微下降（0.382 → 0.371）。原因是 top-k 检索中并非所有原则都相关，直接内化引入了噪声。

### 8. 训练动态分析

- **动作频率演化**：推理步数和搜索次数先增后收敛（约 6 次推理、4 次知识搜索、2 次经验搜索）
- **原则质量提升**：后期生成原则的平均分数（0.500）高于早期（0.462），说明 Agent 的蒸馏能力随训练改善

### 9. 模型规模泛化性

| 模型 | 平均分 |
|------|--------|
| 0.5B | 0.150 |
| 1.5B | 0.270 |
| 3B | 0.382 |
| 7B | 0.417 |

性能随基座模型规模单调增长，证明 EvolveR 是一个可泛化的范式，而非针对特定规模的技巧。

## 优点与局限

### 优点

1. **闭环自演化**：首次实现了"交互→蒸馏→策略更新"的完整闭环，而非单点改进
2. **认知对齐**：自蒸馏在 3B+ 规模超越教师蒸馏，避免了对更强外部模型的依赖
3. **经验库管理完善**：去重、评分、剪枝三级质量控制，确保知识库精炼而非膨胀
4. **即插即用**：作为通用框架可应用于不同规模的基座模型
5. **高效检索**：5 万条原则下延迟仅 0.20s，实用性强
6. **全面消融**：多层次消融实验清晰揭示了每个组件的贡献

### 局限

1. **蒸馏质量受限于基座模型**：小模型（0.5B）自蒸馏效果显著差于教师蒸馏，存在"能力天花板"
2. **仅验证 QA 任务**：未在具身交互、创意生成等其他任务类型上验证
3. **长期遗忘问题未解决**：虽然有剪枝机制，但真正的终身学习（数千任务）的可扩展性未验证
4. **经验内化失败**：尝试将原则直接内化到参数中未成功，说明当前方案的信息传递仍然是"检索式"而非"内化式"
5. **安全性担忧**：Agent 自行演化原则可能发展出不良策略，需要与价值对齐技术配合
6. **冷启动依赖外部模型**：初始 SFT 数据需要 GPT-4o 生成
