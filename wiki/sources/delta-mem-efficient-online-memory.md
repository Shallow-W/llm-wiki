# δ-mem: Efficient Online Memory for Large Language Models

## 一句话总结

提出 δ-mem：在冻结的全注意力骨干网络上，用一个极其紧凑（8×8）的在线关联记忆状态矩阵，通过 delta-rule 学习持续更新，并以低秩修正的方式直接参与注意力计算，实现轻量高效的 LLM 在线记忆。

## 研究背景与动机

### LLM 的记忆困境

随着 LLM 被部署到长期助手和 Agent 系统中，模型需要**持续积累、更新和复用历史信息**。当前的主流方案——简单扩展上下文窗口——存在三大问题：

1. **计算开销大**：标准注意力的计算量随上下文长度呈二次增长。
2. **上下文退化/腐烂（Context Rot）**：即使上下文窗口扩展到百万 token，模型也不一定能有效利用这些额外信息（Hong et al., 2025; Du et al., 2025）。
3. **多轮对话中的迷失**：LLM 在多轮交互中容易丢失关键信息（Laban et al., 2025）。

> **核心洞察**：扩展上下文窗口只是把记忆问题转化为长上下文处理问题，并没有从根本上解决记忆。

### 现有记忆机制的三类范式

论文从两个维度——**记忆状态**（如何存储）和**记忆引导**（如何影响推理）——统一分析现有方法：

| 范式 | 代表方法 | 原理 | 局限 |
|------|----------|------|------|
| **文本记忆（TMM）** | BM25 RAG、LLMLingua-2、MemoryBank | 将记忆存为文本，通过输入上下文注入 | 受上下文窗口限制、检索噪声、压缩信息损失 |
| **外部通道记忆（OMM）** | Memorizing Transformers、MLP Memory | 在外部模块存储记忆，通过检索/编码回传 | 额外开销、融合复杂、与骨干不匹配 |
| **参数化记忆（PMM）** | LoRA、Prefix-Tuning、ROME/MEMIT | 将记忆编码到参数中 | 静态的，无法随序列动态演化 |

**δ-mem 的定位**：参数化记忆的低秩接口 + 在线动态状态 = 既紧凑又可演化。

## 核心方法

### 设计哲学

δ-mem 的核心思路可以概括为一句话：

> **将历史信息压缩到一个固定大小的在线状态矩阵中，通过 delta-rule 不断更新，然后在生成时用状态的读出信号对注意力计算做低秩修正。**

这个过程不修改骨干网络参数，不添加额外的上下文 token，不依赖外部检索模块。

### 计算流程（精读版）

#### 整体架构：δ-mem 在 Transformer 中的位置

要理解 δ-mem 的计算流程，首先需要搞清楚它"插在 Transformer 的哪里"：

```
Transformer 第 ℓ 层的完整计算：

输入 x_t (位置 t 的隐藏状态)
  │
  ├─── [冻结骨干] 标准注意力路径 ──────────────────┐
  │    Q = W^Q · x_t   K = W^K · x_t   V = W^V · x_t  │
  │    Attn = softmax(Q · K^T / √d) · V              │
  │                                                    │
  ├─── [δ-mem] 记忆路径（附加在旁边）──────────────┐    │
  │    ① 投影到记忆空间                              │    │
  │    ② 从 S_{t-1} 读取 r_t                        │    │
  │    ③ 生成修正 Δq_t, Δo_t                        │    │
  │    ④ 修正 Q 和 Attn Output                      │    │
  │    ⑤ 将当前信息写入 S_t                          │    │
  │                                                  │    │
  │ ← 低秩修正加到注意力计算上 →                     ←───┘
  │                                                  
  输出 y_t → 传给下一层
```

**关键理解**：δ-mem 不是替代注意力，而是在冻结的标准注意力旁边加了一条"记忆旁路"。它不改变骨干的权重，只通过给 query 和 output 加偏移量来"引导"注意力的方向。

#### 符号表：每个变量是什么

在深入之前，先把所有关键变量搞清楚：

| 符号                       | 维度                        | 含义                                       | 直觉理解                                    |
| ------------------------ | ------------------------- | ---------------------------------------- | --------------------------------------- |
| $t$                      | 标量                        | 当前处理的 token 位置（1, 2, 3, ...）             | 时间步，类似"第几个字"                            |
| $d$                      | 标量                        | 骨干模型的隐藏维度（如 Qwen3-4B 中 d=2560）           | 每个 token 被表示为一个 d 维向量                   |
| $r$                      | 标量                        | 记忆空间的维度（论文中 **r=8**）                     | 把 d 维压缩到 8 维来存记忆——极致压缩                  |
| $x_t$                    | $\mathbb{R}^d$            | 位置 $t$ 处 Transformer 第 $\ell$ 层的**隐藏状态** | 这就是当前 token 经过前面所有层处理后的"语义表示"——一个 d 维向量 |
| $S_t$                    | $\mathbb{R}^{r \times r}$ | 处理完位置 $t$ 后的**在线记忆状态矩阵**                 | 一个 8×8 的小矩阵，是整个机制的"大脑"——它压缩存储了所有历史信息的关联 |
| $S_{t-1}$                | $\mathbb{R}^{r \times r}$ | 处理位置 $t$ **之前**的记忆状态                     | 上一时刻的记忆——包含了位置 1 到 t-1 所有历史信息的压缩        |
| $q_t^m$                  | $\mathbb{R}^r$            | 位置 $t$ 的**记忆查询向量**                       | "我现在想从记忆里查什么？"——用当前 token 的语义去检索记忆      |
| $k_t^m$                  | $\mathbb{R}^r$            | 位置 $t$ 的**记忆键向量**                        | "当前信息应该存到记忆的哪个'抽屉'里？"——定义存储的索引          |
| $v_t^m$                  | $\mathbb{R}^r$            | 位置 $t$ 的**记忆值向量**                        | "当前信息的具体内容是什么？"——定义要存储的值                |
| $r_t$                    | $\mathbb{R}^r$            | 从记忆状态中**读取到的信号**                         | "记忆里有没有和我当前问题相关的信息？"——查询结果              |
| $\beta_t$                | $\mathbb{R}^r$            | **写入门控**（每个维度一个值）                        | "我有多少新信息需要写入？"——值越大，写入越强                |
| $\lambda_t$              | $\mathbb{R}^r$            | **遗忘门控**（$\lambda_t = 1 - \beta_t$）      | "我要保留多少旧记忆？"——值越大，旧记忆保留越多               |
| $\Delta q_t$             | $\mathbb{R}^d$            | 对注意力 **query 的修正量**                      | "记忆告诉我，我的 query 应该往哪个方向调整？"             |
| $\Delta o_t$             | $\mathbb{R}^d$            | 对注意力 **output 的修正量**                     | "记忆告诉我，注意力的输出应该怎么修正？"                   |
| $\alpha$                 | 标量                        | 修正的**缩放因子**（论文中 α=16）                    | 控制记忆信号对注意力的影响强度                         |
| $W_q^m, W_k^m, W_v^m$    | $\mathbb{R}^{r \times d}$ | 隐藏状态到记忆空间的**投影矩阵**（可训练）                  | 把 d 维的隐藏状态"翻译"到 8 维记忆空间                 |
| $W_q^\Delta, W_o^\Delta$ | $\mathbb{R}^{d \times r}$ | 记忆信号到注意力修正的**映射矩阵**（可训练）                 | 把 8 维记忆信号"翻译"回 d 维来修正注意力                |

> **维度链**：$x_t$（d=2560维）→ 投影到 $q_t^m, k_t^m, v_t^m$（r=8维）→ 操作 $S_t$（8×8）→ 读出 $r_t$（8维）→ 映射到 $\Delta q_t, \Delta o_t$（d=2560维）→ 修正注意力。这就是"压缩→操作→解压"的过程。

---

现在逐步走一遍完整计算。每个位置 $t$ 的计算按以下顺序执行：**投影 → 读 → 引导注意力 → 写**。

---

#### 第一步：记忆投影（Memory Projections）—— 把当前 token "翻译"到记忆空间

骨干 Transformer 给出了当前 token 的隐藏状态 $x_t$（一个 d 维向量）。δ-mem 需要把它"翻译"到一个低维的记忆空间（r=8 维），才能和 8×8 的状态矩阵交互。

$$q_t^m = \text{L2-norm}(\tanh(W_q^m \cdot x_t))$$
$$k_t^m = \text{L2-norm}(\tanh(W_k^m \cdot x_t))$$
$$v_t^m = W_v^m \cdot x_t$$

**逐步拆解（以 $q_t^m$ 为例）**：
1. $W_q^m \cdot x_t$：矩阵乘法，$\mathbb{R}^{8 \times 2560} \times \mathbb{R}^{2560} = \mathbb{R}^{8}$。把 2560 维压到 8 维。
2. $\tanh(\cdot)$：逐元素 tanh，将值压缩到 [-1, 1]，防止数值爆炸。
3. $\text{L2-norm}(\cdot)$：除以 L2 范数，使向量长度归一化为 1。**为什么？** 因为 $S_t$ 会随着序列不断更新，如果 key/query 的数值范围不固定，长序列中状态的尺度会漂移，导致不稳定。

**三个向量的角色**：
- $q_t^m$（查询）：稍后用来**检索**记忆——"记忆里有没有和我相关的信息？"
- $k_t^m$（键）：稍后用来**定位**写入——"我应该把当前信息写到记忆的哪个位置？"
- $v_t^m$（值）：稍后用来**写入**内容——"我要存储的具体内容是什么？"

**注意**：$q_t^m$ 和 $k_t^m$ 做了 L2 归一化（单位向量），$v_t^m$ 没有做——因为 value 需要保留原始幅度来携带信息。

**门控信号**——控制"记住多少，忘掉多少"：

$$\beta_t = \sigma(W_\beta \cdot x_t + b) \in \mathbb{R}^r$$
$$\lambda_t = 1 - \beta_t \in \mathbb{R}^r$$

其中 $\sigma$ 是 sigmoid 函数（输出范围 [0, 1]）。

**直觉理解**：
- $\beta_t$ 的每个维度是一个 [0,1] 的值，表示"当前 token 在这个维度上需要写入多少新信息"
- $\lambda_t = 1 - \beta_t$，表示"在这个维度上保留多少旧记忆"
- 当 $\beta_t[i] \approx 1$：激进写入新信息，几乎覆盖旧记忆
- 当 $\beta_t[i] \approx 0$：保守保留旧记忆，几乎不写入
- **维度级独立控制**：不同维度可以有不同的读写策略——某些维度专注于记住人名，某些专注于记住时间，某些专注于记住情感...

---

#### 第二步：从记忆状态中读取（Reading）—— "记忆里有没有相关的信息？"

$$r_t = S_{t-1} \cdot q_t^m$$

**逐步拆解**：
1. $S_{t-1} \in \mathbb{R}^{8 \times 8}$：这是在处理当前 token 之前，已经累积了所有历史信息（位置 1 到 t-1）的记忆状态。
2. $q_t^m \in \mathbb{R}^{8}$：当前 token 的记忆查询向量（"我想查什么？"）。
3. $S_{t-1} \cdot q_t^m = \mathbb{R}^{8 \times 8} \times \mathbb{R}^{8} = \mathbb{R}^{8}$：矩阵乘向量，得到 $r_t \in \mathbb{R}^{8}$。

**直觉理解**：把 $S$ 想象成一个查找表。$q_t^m$ 是你的查询，$r_t$ 是返回的结果。$S$ 存储的是"key → value"的关联模式——当你用一个查询去乘它时，它会返回与你查询最相关的值组合。

**关键性质**：
- **代价与历史长度无关**：无论之前处理了 100 个 token 还是 100 万个 token，$S_{t-1}$ 始终是 8×8，乘以 8 维向量只需 64 次乘加。这就是"固定大小状态"的优势。
- **与标准注意力互补**：标准注意力是用当前 query 去和所有显式上下文中的 key 做比较；δ-mem 是直接从压缩状态中获取连续信号，不需要逐个比较。

---

#### 第三步：通过低秩修正引导注意力（Steering Attention）—— "记忆信号如何影响推理？"

这一步是将记忆信号 $r_t$（8 维）"翻译"回骨干的维度（d=2560 维），用来修正注意力的计算。

**3a. 生成修正量**

$$\Delta q_t = W_q^\Delta \cdot r_t \in \mathbb{R}^{d}$$
$$\Delta o_t = W_o^\Delta \cdot r_t \in \mathbb{R}^{d}$$

- $W_q^\Delta \in \mathbb{R}^{d \times r}$：把 8 维信号映射回 2560 维的 query 修正。
- $W_o^\Delta \in \mathbb{R}^{d \times r}$：把 8 维信号映射回 2560 维的 output 修正。

**直觉**：$r_t$ 是从记忆中提取的 8 个数字——非常紧凑的"信号"。$\Delta q_t$ 和 $\Delta o_t$ 是将这些信号展开为 2560 维的修正向量，告诉注意力"你的 query 应该往哪个方向偏"和"你的输出应该怎么调整"。

**3b. 修正注意力的 query**

标准骨干先计算原始 query：
$$q'_t = W^Q \cdot x_t$$
这是冻结骨干的正常计算，$\delta$-mem **不碰** $W^Q$。

然后在 query 上加上记忆修正：
$$\tilde{q}_t = q'_t + \frac{\alpha}{r} \cdot \Delta q_t$$

- $\frac{\alpha}{r}$：缩放因子。$\alpha = 16$，$r = 8$，所以实际缩放 $= 16/8 = 2$。
- **效果**：修正后的 query $\tilde{q}_t$ 同时包含了两部分信息：(1) 当前 token 的语义（来自骨干），(2) 历史上下文的记忆信号（来自 δ-mem）。

**3c. 用修正后的 query 做注意力**

$$a_t = \text{Attn}(\tilde{q}_t, K_{\leq t}, V_{\leq t})$$

注意！$K_{\leq t}$ 和 $V_{\leq t}$ 是冻结骨干产生的原始 key 和 value，**没有被修改**。只有 query 被修正了。

> **为什么只修正 query 不修正 key？** 因为修正 query 等于改变了"我在找什么"，这足以让注意力聚焦到不同的位置。而修改所有历史 token 的 key 则代价高昂（需要改所有位置）。

**3d. 修正注意力的 output**

$$\tilde{y}_t = a_t + \frac{\alpha}{r} \cdot \Delta o_t$$

在注意力输出 $a_t$ 上再加一个 output 侧的修正。这允许记忆信号在注意力计算之后再次施加影响。

**完整信号流**：
```
x_t (d=2560)
  │
  ├──→ W^Q · x_t = q'_t (原始 query)
  │                      │
  │                      ├── + (α/r) · Δq_t  ←── W_q^Δ · r_t ←── S_{t-1} · q_t^m
  │                      │
  │                      ↓
  │                  q̃_t (修正后 query)
  │                      │
  │                      ↓
  │            Attn(q̃_t, K, V) = a_t
  │                                    │
  │                                    ├── + (α/r) · Δo_t  ←── W_o^Δ · r_t
  │                                    │
  │                                    ↓
  │                                ỹ_t (最终输出)
  │
  └──→ 同时：k_t^m, v_t^m, β_t 准备写入 S_t
```

> **关键区别**：虽然 $W_q^\Delta$ 和 $W_o^\Delta$ 训练后固定，但输入 $r_t$ 来自动态状态 $S_{t-1}$。同一组参数在不同历史下产生不同的引导效果——这不是静态 adapter，而是**状态条件化的动态修正**。这就是为什么论文说这是"low-rank correction"而非"low-rank adaptation"。

---

#### 第四步：通过 Delta-Rule 写入状态（Writing）—— "把当前信息压缩存入记忆"

注意力计算完成后，δ-mem 将当前 token 的信息写入状态矩阵 $S$，为后续位置的记忆做准备。

**4a. 预测：旧状态对当前 key 的预测是什么？**

$$\hat{v}_t = S_{t-1} \cdot k_t^m$$

这是旧记忆状态 $S_{t-1}$ 对"当前 key $k_t^m$ 应该关联什么 value"的**预测**。

**直觉**：如果之前的记忆已经学好了这个关联，那么 $\hat{v}_t \approx v_t^m$（预测接近真实值），说明"这个信息我已经记住了"。

**4b. 计算残差（delta）**

$$\delta_t = v_t^m - \hat{v}_t = v_t^m - S_{t-1} \cdot k_t^m$$

这就是"**delta**"——预测值和真实值之间的差距。它衡量了"还有多少新信息是记忆中没有的"。

**4c. 完整的门控 delta-rule 更新**

$$S_t = \text{Diag}(\lambda_t) \cdot S_{t-1} + \text{Diag}(\beta_t) \cdot \underbrace{(v_t^m - S_{t-1} \cdot k_t^m)}_{\text{残差 } \delta_t} \cdot (k_t^m)^\top$$

展开后可以看到三个清晰的项：

$$S_t = \underbrace{\text{Diag}(\lambda_t) \cdot S_{t-1}}_{\text{① 保留项}} - \underbrace{\text{Diag}(\beta_t) \cdot S_{t-1} \cdot k_t^m \cdot (k_t^m)^\top}_{\text{② 擦除项}} + \underbrace{\text{Diag}(\beta_t) \cdot v_t^m \cdot (k_t^m)^\top}_{\text{③ 写入项}}$$

**逐项解释**：

| 项 | 公式 | 维度变换 | 含义 |
|-----|------|----------|------|
| **① 保留** | $\text{Diag}(\lambda_t) \cdot S_{t-1}$ | diag(r) × (r×r) = r×r | 旧状态按维度衰减保留。$\lambda_t \in [0,1]^r$，每个维度独立控制保留比例。如果某维度 $\lambda=0.9$，旧记忆保留 90%。 |
| **② 擦除** | $\text{Diag}(\beta_t) \cdot S_{t-1} \cdot k_t^m \cdot (k_t^m)^\top$ | diag(r) × (r×r) × r × r^⊤ = r×r | 移除旧状态中沿当前 key 方向的旧预测。如果之前记忆中"张三 → 北京"，现在要更新为"张三 → 上海"，这步先把"北京"擦掉。 |
| **③ 写入** | $\text{Diag}(\beta_t) \cdot v_t^m \cdot (k_t^m)^\top$ | diag(r) × r × r^⊤ = r×r | 将新的 value 写入当前 key 方向。写入"张三 → 上海"。 |

> **维度级展开**（第 $i$ 行）：
> $$s_t^{(i)} = \lambda_{t,i} \cdot s_{t-1}^{(i)} + \beta_{t,i} \cdot (v_{t,i}^m - s_{t-1}^{(i)} \cdot k_t^m) \cdot (k_t^m)^\top$$
>
> 这说明每个记忆维度 $i$ 有自己独立的 $\lambda_{t,i}$ 和 $\beta_{t,i}$。不同维度可以同时做不同的事情：某些维度在"死记硬背"新信息（$\beta$ 大），某些维度在"温故知新"保留旧记忆（$\lambda$ 大）。

**4d. Delta-rule 的核心直觉**

> 把 $S$ 想象成一本笔记。每遇到一个新信息 $(k_t^m, v_t^m)$：
> 1. 先翻到 $k_t^m$ 对应的页，看看之前记了什么（$\hat{v}_t = S_{t-1} \cdot k_t^m$）
> 2. 如果记的和新信息一样（$\hat{v}_t \approx v_t^m$），那**什么都不用做**
> 3. 如果不一样，只把**差的部分**写入（$v_t^m - \hat{v}_t$）
>
> 这就是 delta-rule——"只记住你不知道的"。数学上，它等价于对损失 $\mathcal{L}(S) = \frac{1}{2}\|S \cdot k_t^m - v_t^m\|^2$ 做一步在线 SGD。

> **与 LoRA 的本质区别**：LoRA 的低秩更新 $\Delta W = B \cdot A$ 在训练结束后就固定了——所有输入共享同一组修正。δ-mem 的低秩修正 $\Delta q_t = W_q^\Delta \cdot (S_{t-1} \cdot q_t^m)$ 中，$W_q^\Delta$ 虽然固定，但中间的 $S_{t-1}$ 是随序列不断演化的。所以同样是"低秩"，LoRA 是静态的，δ-mem 是**状态条件化动态的**。

### 三种写入粒度策略

论文研究了三种写入策略，它们共享同一个 delta-rule 更新公式，但在"什么时机写入"和"写入什么"上有根本性区别。如图 1 所示，TSW 在每个 token 写入，SSW 在每个消息段写入（段内平均），MSW 写入多个并行子状态后拼接读出。

#### Token-State Write（TSW，Token 级写入）

$$S_t = \text{Update}(S_{t-1}, x_t)$$

- **写入粒度**：每遇到一个 token 就执行一次完整的状态更新（Read→Steering→Write 全链路）
- **核心优点**：保留最细粒度的信息，能捕捉到对话中微妙的语气转折、token 级别的语义变化
- **核心缺点**：
  - 格式符号（如 `<|im_start|>`、换行符、缩进）也会触发写入，引入噪声
  - 重复表达（如"嗯嗯""好的好的"）会导致冗余写入
  - 短期波动（如语气词）会让状态演化过于颠簸
- **适用场景**：需要捕捉局部快速变化的场景；在 Qwen3-4B 上效果最优（总平均 51.66%）

#### Sequence-State Write（SSW，段级写入）

首先对第 $j$ 条消息（或语义段）内的所有 token 隐藏状态取平均：

$$\bar{x}^{(j)} = \frac{1}{|M(j)|} \sum_{t \in M(j)} x_t$$

然后用这个平均表示更新状态（每条消息只更新一次）：

$$S^{(j)} = \text{Update}(S^{(j-1)}, \bar{x}^{(j)})$$

- **写入粒度**：每条消息（或每个语义段）只写入一次，用段内所有 token 的平均隐藏状态作为写入内容
- **核心优点**：
  - **减少冗余**：一整条消息的信息被压缩为一次写入，避免重复 token 的反复扰动
  - **平滑演化**：平均操作天然起到低通滤波效果，状态变化更稳定
- **核心缺点**：段内 token 级别的细节（如特定措辞、局部强调）被平均掉了
- **适用场景**：对话和 Agent 轨迹等"消息是天然语义单元"的场景；在 Qwen3-8B 上效果最优（总平均 50.86%）

> **TSW vs SSW 的直觉对比**：想象你在做会议记录。TSW 是每听到一个词就记一笔，SSW 是等一个人说完一整段话再记一个摘要。前者细节丰富但容易乱，后者简洁但可能遗漏细节。

#### Multi-State Write（MSW，多状态并行写入）

将单一状态分解为 $N$ 个并行子状态（论文默认 $N=4$），每个子状态独立演化：

$$S_t = \{S_t^{(1)}, \ldots, S_t^{(N)}\}$$

$$S_t^{(i)} = \text{Update}^{(i)}(S_{t-1}^{(i)}, x_t) \quad \text{for } i = 1, \ldots, N$$

读出时，每个子状态独立产生 read 向量，然后拼接：

$$r_t = \text{Concat}(r_t^{(1)}, \ldots, r_t^{(N)})$$

其中 $r_t^{(i)} = S_{t-1}^{(i)} \cdot q_t^{m,(i)}$

- **核心设计动机**：单一 8×8 状态需要同时容纳**事实、偏好、任务进度、局部事件**等多种信息，容易发生覆盖和干扰。MSW 让每个子状态专精一类信息
- **核心优点**：
  - **减少干扰**：不同类型信息写在不同"笔记本"上，不会互相覆盖
  - **小模型增益显著**：SmolLM3-3B 上 MSW 将平均分从 26.08% 提升到 36.96%（+10.88），是三种策略中提升最大的
- **核心缺点**：参数开销略大（19.47M vs 4.87M），且需要为每个子状态配独立的 $W_q^\Delta$ 和 $W_o^\Delta$
- **实验表现**：在 LoCoMo 长对话基准上 MSW 最优（49.12%），在 MemoryAgentBench 上也最强（38.85%）

#### 三种策略的实验对比

| 策略 | Qwen3-4B 最佳 | Qwen3-8B 最佳 | SmolLM3-3B 最佳 | 记忆密集型任务 |
|------|---------------|---------------|-----------------|---------------|
| TSW | ✅ 51.66% | — | — | 中等 |
| SSW | 51.44% | ✅ 50.86% | — | 中等 |
| MSW | 50.74% | — | ✅ 36.96% | ✅ 最强 |

**模型规模规律**：
- **小模型（3B）**：MSW 最好（+10.88）。小模型的 8×8 状态容量有限，单状态难以同时容纳多种信息，分离子状态显著减少干扰
- **中等模型（4B）**：TSW 最好。模型已有一定容量，token 级粒度能捕捉足够细节
- **大模型（8B）**：SSW 最好。大模型本身推理能力强，不需要 token 级噪声，段级平均更稳定

### 训练目标

δ-mem 使用**标准 SFT 交叉熵损失**，不需要特殊的记忆训练目标。这是 δ-mem 的一个重要设计选择——记忆能力是从普通监督学习中"涌现"的，而非通过专门设计的记忆损失来强制。

#### 训练数据与设置

- **训练数据**：QASPER 数据集最短 2,219 样本分割（Dasigi et al., 2021），最长序列 8,269 tokens
- **训练 epoch**：1 个 epoch
- **最大序列长度**：骨干训练 512 tokens，记忆写入预算 8,192 tokens
- **超参数**：$r=8$，$\alpha=16$，默认 qo 分支注入，MSW 子状态数 $N=4$
- **硬件**：8 × A800 GPU，bfloat16 精度，DeepSpeed ZeRO-2
- **优化器**：fused AdamW，峰值学习率 $2 \times 10^{-4}$，cosine decay，warmup 比例 0.1
- **批次**：每设备 batch size 1，4 步梯度累积，有效全局 batch size 32

#### 损失函数

对于每个训练样本，执行以下流程：

**第一步：上下文写入状态**

上下文 token（如历史对话、参考文档）被逐 token（或逐段）喂入 δ-mem，执行 Read→Steering→Write 全链路，最终产生压缩记忆状态 $S_C$。注意：**这些上下文 token 不作为显式输入回传给冻结骨干**——它们只通过 δ-mem 的状态间接影响骨干。

> 这等价于"先读一遍材料做笔记（$S_C$），然后合上材料只凭笔记答题"。

**第二步：Query + Response 训练**

冻结骨干只接收 query $Q$ 和 response $Y$，但注意力被 $S_C$ 引导：

$$\mathcal{L}_{SFT} = -\sum_{j=1}^{|Y|} \log p_{\phi,\theta}(y_j \mid Q, y_{<j}, S_C)$$

**符号说明**：

| 符号 | 含义 |
|------|------|
| $\phi$ | 冻结骨干参数（不参与梯度更新） |
| $\theta$ | δ-mem 的可训练参数（$W_q^m, W_k^m, W_v^m, W_\beta, W_q^\Delta, W_o^\Delta$） |
| $Q$ | 查询/问题（如"根据上文，用户喜欢什么？"） |
| $Y$ | 目标回答（如"用户喜欢科幻电影"） |
| $y_{<j}$ | 回答中第 $j$ 个 token 之前的所有 token（自回归） |
| $S_C$ | 上下文写入后产生的压缩记忆状态 |

**关键设计**：

1. **骨干完全冻结**：$\phi$ 不参与梯度计算，确保通用能力不丢失
2. **记忆从 SFT 涌现**：没有单独的记忆损失（如对比学习、重建损失），记忆能力完全来自"让模型用记忆状态生成正确回答"这一任务压力
3. **上下文隐式化**：上下文只写入 $S_C$，不进入骨干的自回归计算。这强迫 δ-mem 必须将上下文信息有效编码到 8×8 状态中，否则模型无法正确回答

> **为什么这个设计有效？** 如果把上下文直接作为显式输入（vanilla），模型可以"作弊"——直接看上下文找答案。δ-mem 的设计强迫模型必须把上下文压缩进 8×8 状态，然后凭这个状态答题。如果 8×8 状态没记住关键信息，模型就会答错——梯度就会迫使 δ-mem 学习更好的压缩策略。这是一种"倒逼学习"。

## 优点与局限

### 优点

1. **极致紧凑**：8×8 状态矩阵（仅 4.87M 可训练参数，0.12%）即可实现有效记忆
2. **骨干冻结**：不需要修改或替换骨干网络，兼容任意全注意力 Transformer
3. **在线演化**：状态随序列动态更新，而非训练后固定
4. **通用保持**：在增强记忆的同时基本不损害通用能力
5. **与注意力深度耦合**：记忆直接参与前向计算，而非通过外部检索路径

### 局限与开放问题

1. **状态容量有限**：8×8 矩阵的容量有上限，超长历史如何处理？
2. **训练数据依赖**：仅在 QASPER 的 2,219 样本上微调一个 epoch，泛化性有待进一步验证
3. **写入策略选择**：TSW/SSW/MSW 的最优选择依赖骨干模型大小，缺乏统一的自动选择机制
4. **仅实验了 SFT 训练**：是否可以与 RLHF/DPO 等对齐方法结合？
5. **与长上下文窗口的互补性**：论文没有探索 δ-mem 与长上下文窗口的联合使用效果

## 原始笔记

### 论文的方法-直觉映射

理解 δ-mem 的一个好类比是"**带有遗忘机制的快速缓存**"：
- 就像 CPU 的 L1 缓存不需要存储所有内存数据一样，δ-mem 不需要存储所有历史 token
- 它存储的是"**关联**"（key→value 的映射），而非原始文本
- Delta-rule 本质上是"**预测误差驱动的学习**"：已经记住的不重复存储，只有预测错误的部分才写入
- 遗忘门（λ）类似于缓存替换策略，控制哪些旧信息需要淘汰

### 与 MCP/LSP 的关联思考

δ-mem 解决的是 LLM 内部的"记忆"问题，而 [[Model-Context-Protocol-MCP|MCP]] 解决的是 LLM 与外部世界的"交互"问题。两者互补：
- δ-mem：LLM 如何在内部压缩和复用历史信息
- MCP：LLM 如何获取外部工具和数据
- 如果 Agent 同时需要内部记忆和外部工具，δ-mem + MCP 可能是一个强大的组合

### 数学优雅性

Delta-rule update 的形式 $S_t = \lambda_t S_{t-1} + \beta_t(v_t - S_{t-1}k_t)k_t^\top$ 非常优雅：
- 当 $S_{t-1}k_t \approx v_t$ 时（已记住），更新趋近于零
- 当 $S_{t-1}k_t \neq v_t$ 时（预测错误），更新量正比于误差
- 这本质上是在线 SGD 一步，优化的是 $\|Sk_t - v_t\|^2$

## 关联

- 与 [[Context2LoRA]] 的对比：都是低秩修正，但 δ-mem 是动态的，Context2LoRA 是静态的
- 与 [[Memorizing-Transformers]] 的对比：都用外部记忆，但 δ-mem 用固定大小状态，而非无限增长的 kNN 存储
- 与 [[Titans]] 的对比：都关注测试时记忆，但 δ-mem 不替换骨干，而是附加在冻结骨干上
- 与 [[mem0-building-production-ready-ai-agents|Mem0]] 的对比：δ-mem 是参数级内部记忆（8×8 状态矩阵），Mem0 是文本级外部记忆（向量检索+图结构）；两者互补——δ-mem 解决"怎么记"，Mem0 解决"存什么"和"怎么找"
- [[Context-Rot]] 解释了为什么简单扩展上下文窗口不够
- [[LLMs-Get-Lost-in-Multi-turn]] 说明了多轮对话中记忆退化的现象

| 9   | **Kimi Linear: An Expressive, Efficient Attention Architecture** (Team et al., 2025)                 | 线性注意力架构的最新进展，δ-mem 的遗忘门设计受其启发。                                     |
| 10  | **Mem0: Building Production-ready AI Agents with Scalable Long-term Memory** (Chhikara et al., 2025) | 工业级的 Agent 记忆系统，从工程实践角度解决记忆问题，与 δ-mem 的学术角度互补。                     |
