Joint Deployment and Request Routing for
Microservice Call Graphs in Data Centers
Yi Hu ,H a oW a n g , Liangyuan Wang , Menglan Hu ,K a iP e n g ,
and Bharadwaj Veeravalli , Senior Member, IEEE
Abstract —Microservices are an architectural and organizational
paradigm for Internet application development. In cloud data
centers, delay-sensitive applications receive massive user requests,
which are fed into multiple queues and subsequently served by
multiple microservice instances. Accordingly, effective deployment
of multiple queues and containers can signiﬁcantly reduce queuing
delay, processing delay, and communication delay. Due to the in-
creased complexity of call dependencies and probabilistic routing
paths, the deployment of service instances fully interacts with
request routing, bringing great difﬁculties to service orchestra-
tion. In this case, it is valuable to simultaneously consider service
deployment and request routing in a ﬁne-grained manner. How-
ever, most existing studies considered them as two independent
components with local optimization, while data dependencies and
the instance-level deployment are ignored. Therefore, this paper
proposes to jointly optimize the deployment and request routing
of microservice call graphs based on ﬁne-grained queuing network
analysis and container orchestration. We ﬁrst formulate the prob-
lem as a mixed-integer nonlinear program and exploit open Jack-
son queuing networks to model intrinsic data dependencies and
analyze response latency. To optimize the overall cost and latency,
this paper presents an efﬁcient two-stage heuristic algorithm, which
consists of a resource-splitting-based deployment approach and a
partition-mapping-based routing method. Further, this paper also
provides mathematical analysis on the performance and complexity
of the proposed algorithm. Finally, comprehensive trace-driven
experiments demonstrate that the overall performance of our ap-
proach is better than existing microservice benchmarks. The aver-
age deployment cost is reduced by 27.4% and end-to-end response
latency is reduced by 15.1% on average.
Index Terms —Cloud data centers, microservice call graphs,
queuing theory, request routing, service deployment, services
computing.

Digital Object Identiﬁer 10.1109/TPDS.2023.3311767I. I NTRODUCTION
MICROSERVICES are an architectural and organizational
approach where applications are structured as a col-
lection of small independent units or services communicating
through well-deﬁned interfaces [1]. The loosely-coupled and
independently-deployable microservices are gradually replac-
ing traditional monolithic applications [2],[3]. In contrast,
microservices are highly maintainable, scalable and easy to
develop, accelerating the innovation and evolution for new
features.
Nowadays, many cloud service providers such as AWS and
Azure have deployed numerous microservices to support var-
ious real-time cloud applications. In cloud data centers, these
delay-sensitive applications receive a large number of user re-
quests in every second, which are fed into multiple queues and
subsequently served by multiple microservice instances [4],[5].
The user requests are processed by a set of microservices which
recursively call and frequently communicate with each other,
thereby forming a microservice call graph [6]. The microservice
multiplexing will occur when one microservice is present in mul-
tiple microservice call graphs. The same type of microservice
containers can be shared by different user requests, which results
in multi-instance and geographically distributed microservice
deployment. Hence, the multiple queues and containers for the
call graphs can be modeled and analyzed by queuing networks.
In this case, efﬁcient deployment of multiple queues and con-
tainers can effectively reduce queuing delay, processing delay,
and communication delay, which brings great opportunities to
service performance optimization.
However, due to the increased complexity of call dependen-
cies and microservice multiplexing, the deployment of service
instances fully interacts with request routing, raising great chal-
lenges to service orchestration. On one hand, the effect of service
deployment highly depends on the ﬁne-grained microservice
processing and massive requests routed among service instances.
Microservice multiplexing and container sharing occur in large
numbers, which allow different user requests to utilize the same
microservice containers. At the same time, user requests traverse
certain microservices in a speciﬁc order to complete a service,
and the frequent communication of adjacent microservice pairs
is known at a coarse-grained level, which affect the deployment
strategy. On the other hand, request routing requires deployed
service instances as a precedent and heavily relies on the
instance-level routing strategy for service level objectives [7].

Instances required by microservices can be deployed on different
processors, thus resulting in the multi-instances and multi-queue
situation. A user request passing through a speciﬁc microservice
can be routed to containers of the same microservice. In this case,
containers of the identical microservice across different call
graphs can be shared, which means that the ﬁne-grained request
routing heavily relies on microservice deployment scheme at
the container level. Therefore, to improve the performance of
delay-sensitive applications, it is highly desirable to jointly
optimize microservice deployment and request routing based
on ﬁne-grained queuing network modeling and multi-instance
orchestration.
Nevertheless, existing work neglected to explore efﬁcient
deployment of multiple queues and containers based on queuing
network modeling for simultaneous reduction of queuing delay,
processing delay, and communication delay [8],[9],[10].A l -
though a few previous studies introduced simple queuing models
and optimized end-to-end latency, yet they failed to employ ﬁne-
grained container deployment to reduce queuing latency [11],
[12]. In addition, in most existing work service deployment and
request routing are investigated as two isolated problems while
the intrinsic data dependencies among services are ignored [13],
[14],[15]. Industrial products such as Kubernetes also decou-
ples deployment and routing as different modules or plug-in
components [16]. Such decoupling simpliﬁes the development
and maintenance, but it only supports local optimization in both
components, which is apparently disadvantageous for global
performance optimization. Therefore, due to the absence of
comprehensive research on the joint optimization problem, it
is recommended to exploit ﬁne-grained multi-instance/queue
deployment and capture data dependencies to achieve elegant
microservice orchestration.
Motivated by the aforementioned needs and difﬁculties, this
paper jointly optimizes microservice deployment and request
routing based on queuing network modeling and ﬁne-grained
container orchestration, while three fundamental challenges
are to be solved. First, multi-instance and multi-location ser-
vice deployment decides the mapping of request queues, while
coarse-grained microservice dependencies affect ﬁne-grained
service deployment. Owing to complicated call dependencies
and service multiplexing, the deployment of service instances
fully interacts with request routing, bringing great difﬁculties
to service orchestration. In addition, due to limited processor
resources, different deployment policies may lead to different
performance in queuing latency, processing latency, and com-
munication latency. Accordingly, when deploying microservice
instances, how to determine the trade-off of different latency
becomes the second challenge. Furthermore, since multiple
containers and data dependencies result in complex routing
paths in queuing networks, different routing strategies generate
various forwarding probabilities and multi-queue requests are
reasonably mapped to multiple microservice instances. Thus, to
prevent service failure, how to exactly route user requests via
the accurate forwarding probability at the container level is the
third challenge.
To this end, this paper exploits multi-instance service deploy-
ment and probabilistic request routing in data centers to deliverhigh performance in cost saving, end-to-end delay minimization,
and request success rate maximization. In summary, our work
makes the following contributions:
1) This article formulates the joint microservice deployment
and request routing problem (JMDRP) as a mixed-integer
nonlinear programming (MINLP) model, which captures
the data dependency and instance-level service process-
ing for request ﬂows. Also, microservice multiplexing,
container sharing, and probabilistic routing paths make
service deployment and request routing tightly coupled
and complementary. Moreover, we employ the open Jack-
son queuing network to model and analyze the latency in
request queuing, processing, and communications, which
offers guidelines to ﬁne-grained multi-instance deploy-
ment.
2) Since the targeting problem is NP-hard, this paper pro-
poses a two-stage heuristic for joint deployment and
routing, which illustrates that multi-queue requests from
different call chains are appropriately mapped to multiple
microservice instances. Based on the intuition of factor-
based resource splitting and microservice-level call depen-
dencies, the ﬁrst stage is referred to as the greedy microser-
vice deployment algorithm (GMDA), which implements
ﬁne-grained container deployment to simultaneously re-
duce queuing delay, processing delay, and communication
delay. The second is referred to as the request-matching
and partition-mapping (RMPR) algorithm, which splits
request ﬂows and routes them to microservice instances
in a probabilistic manner, further optimizing the response
latency and request success rate.
3) This paper also provides mathematical analysis on the
performance and complexity of the proposed approach
with complex queuing networks, demonstrating that rea-
sonably integrating microservice instances and request
ﬂows effectively improves overall performance. Finally,
extensive trace-driven simulations are carried out to show
the superior performance of the proposed algorithm by
comparing with existing policies. Extensive simulation
results show that average deployment cost is reduced
by 27.4%, the average response latency is improved by
15.1%, and request success rate is improved by 5.47 %
compared to the benchmark algorithms on average.
The reminder of this paper is organized as follows. Section II
reviews the related work. In Section III, we introduce the math-
ematical model. Section IVpresents the proposed algorithm and
optimization objectives. Complexity analysis and theorems are
offered in Section V. Finally, Section VIprovides the perfor-
mance evaluation, with conclusions following in Section VII.

task scheduling algorithm for distributed green data centers.

B. Request Routing
There are also some studies that have developed heuristic
algorithms to consider the request latency, resource consumption
or network throughput under the multi-objective constraints of
microservice deployment and routing optimization. In [22],Y u
et al. took maximizing the throughput of the service chain as the
optimization goal and proposed an efﬁcient heuristic algorithm
to solve the service chain deployment problem under multi-core
processors. Guo et al. [23] believed that the resource occupation
is related to nodes and link locations, and proposed a service
chain placement method based on dynamic programming to ﬁnd
a near-optimal solution. Zhang et al. [24] proposed a candidate
path-based service chain deployment (CPBSD) algorithm to
solve the joint optimization problem of bandwidth and com-
puting resource utilization. The work of [22],[23],[24] solved
the problem of microservice deployment under multiple target
constraints in a data center, but did not carefully focus on the de-
pendency of neighbouring microservices and routing problems.
Some existing work that optimizes request routing strategies
in a network topology to improve service speed and reduce
response latency has been investigated in detail. Niu et al. [25]
modeled the request routing problem as a non-cooperative game
model, which used the Nash negotiation solution to coordi-
nate resource allocation, and proposed a message queue-based
load balancing algorithm that reduced the average response
time of microservice chains. Mirhosseini et al. [26] proposed a
coarse-grained multi-thread scheduling strategy for the network
architecture, which shortened the service response time. Zhaoet al. [27] proposed a distributed redundant placement frame-
work named Sample Average Approximation (SAA-RP), which
allowed one candidate to be scheduled to multiple sites for faster
response to service requests. Gamal et al. [28] established a
multi-objective evolutionary algorithm based on decomposition
for transmission delay and processing delay, which effectively
reduced the request response latency. The above work did not
consider service deployment, while in this paper we jointly
optimize deployment and request routing at the container level.
C. Joint Microservice Deployment and Routing
It is worth noting that microservice deployment and request
routing are two tightly coupled subproblems. The deployment
scheme of microservice instances decides the routing of request
ﬂows, while the performance of the microservice deployment
can only be reﬂected after the routing is determined. Thus, they
are interact directly and a global optimal solution is desired
by jointly solving two coupled subproblems. The work of Yu
et al. [29] is similar to ours. Responding to the dual objectives
of extending and optimizing the system’s energy consumption,
they proposed a three-stage algorithm to determine the number
of microservice instances and the request routing scheme in
turn, but the transmission delay of requests was ignored and the
optimization of the system response latency was very limited.
Feng et al. [30] proposed a fast approximation algorithm based
on the queuing length, which ﬁrst determined the deployment
area of virtual network functions in a distributed cloud network
and then performed network resource allocation in multiple
clouds. However, this scheme had a high additional cost in terms
of network resources and did not take into account the data inter-
dependencies and communications among microservices. Also,
they neglected to leverage efﬁcient ﬁne-grained deployment of
multiple queues and containers. In contrast,this paper jointly
optimizes the deployment and request routing of microservice
call graphs based on ﬁne-grained queuing network analysis
and container orchestration. We propose a two-stage heuristic
framework to simultaneous reduce queuing delay, processing
delay, and communication delay.

C. Request Models
For each microservice node, the request can be processed at
any container of the microservice. Once a user request reaches
a microservice node, it will be routed to containers of the node
for processing. As shown in Fig. 2, when a request arrives at
microservice instances, it enqueues and waits in the queue until
it is processed. After a microservice node processes the request,
it calls its successor node to process the request. Thus, contain-
ers of a microservice employs remote procedure calls (RPCs)
to call other containers of the successor node to implement
this, which involves data transfer and incurs communication
delay. Accordingly, after a request reaches the system, it is
either processed in containers of a microservice immediately,
or transmitted between two processors, or it stays in a queue
of a processor. We shall allocate sufﬁcient computing resources
for each microservice such that numerous user requests can be
distributed among multiple containers and processed in time,
without saturating the system. In addition, user request ﬂows

Probabilistic request routing allows multiple different routing
paths to exist for each request ﬂow. Accordingly, there are com-
plex routing paths for request ﬂows from different microservice
call chains, and each user request will probabilistically select
these paths. Considering the condition of microservice container
sharing, there exists an intersection between different request
ﬂows, thus resulting in complex dynamics within the overall
queuing system. Thus, the probabilistic routing strategy is criti-
cal in achieving joint optimization of deployment and routing. In
fact, user requests randomly arrive in the system and obey a Pois-
son process. Each request ﬂow will be sequentially processed by
all microservices on the targeting service chain, which incurs the
response latency in a data center. This paper aims to minimize the
long-term average latency of all requests for all microservices
through the joint optimization of microservice deployment and
request routing. Based on the above discussion and assumptions,
this section elaborates on how queuing network theory can be
applied to the request model, and a data center scenario with
multiple request ﬂows is analyzed.

To solve the joint microservice deployment and request rout-
ing problem (JMDRP), we design an efﬁcient two-stage heuris-
tic algorithm to improve the overall performance. Consider a
simpliﬁed version of the problem, where each microservice is
only deployed on one processor with a unique resource capacity.
The simpliﬁed JMDRP is regarded as the combination of a bin
packing problem and number partition problem, which are all
NP-hard [31],[32]. Hence, the JMDRP is NP-hard as well.
Besides, we propose the optimization objectives based on the
above models and relationships between parameters and vari-
ables. Then, we propose the greedy microservice deployment
algorithm (GMDA) for deploying microservice instances in a
ﬁne-grained manner, which minimizes the deployment cost in
the worst case. The effective deployment of multiple queues and
containers also can signiﬁcantly reduce queuing delay and com-
munication delay, boosting overall service performance. Finally,
for the service request routing, we design a request-matching
partition-mapping (RMPR) algorithm to improve the success
rate and further reduce the average response latency, which
enables the efﬁcient routing of request ﬂows and determines
the forwarding probability at the instance-level.

VI. P ERFORMANCE EVA L UAT I O N
In this section, we present trace-driven simulations to evaluate
the performance of the two-stage heuristic algorithm framework
as compared with other baselines. Our experimental setup is
based on real-world traces in data center networks.
A. Experiment Settings
Microservices and call graphs: Luo et al. [6]summarized
a variety of microservice call graphs and modeled them with
tree structures, the vast majority of which are pipelined chains.
For simplicity, we extract and preprocess independent pipelined
request chains from the call graphs. Traced by this article,
the number of microservices is typically scaled from 2 to 50.
The number of requests ranges from 10 to 1000, and each
request traverses a long call chain which consists of at most
14 microservices in our setup, where the length of chain mostly
ranges from 2 to 9.
Network topology and processors: Data center network topol-
ogy (e.g., fat tree) has been investigated in [34],[35]. In our
experiments, a full connection service mesh or connection graph
is considered to model a data center, which contains from 10 to
64 processors. Besides, there is also sufﬁcient switch capacity
and network bandwidth to serve each request. The resource
capacity (the number of cores) per processor is scaled from 8
to 64. The processing core of processors indicates the ability
of handling a unit microservice requests, which serve differ-
ent types of microservices at different speeds. Furthermore, as
microservice-based requests are typically latency-sensitive, we
believe that the maximum communication latency between two
processors should not exceed 5 percent of the user-tolerated
latency in a data center.Distribution of request ﬂow rate and containers: In data center
networks, [36],[37] show that the arrival rate of requests follows
an interval time distribution. In our experiment conﬁgurations, it
is assumed that the arrival rate of request ﬂows follows a Poisson
distribution scaling from 1 to 10. It is worth noting that the
same user request ﬂow can arrive multiple times. Following the
open Jackson queuing theory, each service instance of the same
microservice has the same exponential service rate. Multiple
service requests can be routed to the same containers, and a
request can be routed to different processors and eventually
merged. The service rate per core or container is less than the
average arrival rate since microservice architectures typically
require the deployment of hundreds of containers. To ensure
that the simulation is scalable and large-scale, it is assumed that
processing rate Λmof a single core varies from 1 to 5 for each
microservice.
Baseline algorithms: We choose two robust benchmark al-
gorithms consisting of Random Local Search (RLS) [38] and
modiﬁed First Fit Decreasing (FFD) [39],[40], which are widely
used in resource management and combined with the GMDA in
the experiment. We also implement a variant DRS algorithm [41]
with probabilistic request routing, the modiﬁcation of which is
to accommodate our joint optimization problem. Moreover, a
three-stage algorithm LEGO [29] is used as a baseline for the
joint deployment and routing, which adopts the deterministic
routing policy and balanced deployment. To comprehensively
evaluate the performance of GMDA-RMPR, we compare it with
them and calculate average values as simulation results through
a large number of repeated experiments.

VII. C ONCLUSION
In this paper, we have studied the joint optimization prob-
lem of microservice deployment and request routing based
on ﬁne-grained queuing network modeling and multi-instance
orchestration, which aims to reduce the total cost and end-to-end
response latency, and improve the success rate of requests. To
achieve a trade-off, we have formulated the JMDRP as a MINLP
problem which is NP-hard and employed the open Jackson queu-
ing network to model call dependencies and analyze the response
latency. To this end, this paper proposes a two-stage heuristic for
joint deployment and routing, which illustrates that multi-queue
requests from different call chains are appropriately mapped
to multiple microservice instances. For the multi-instance mi-
croservice deployment, a resource-splitting-based GMDA algo-
rithm is proposed to implement ﬁne-grained container deploy-
ment to simultaneously reduce queuing delay, processing delay,
and communication delay. Furthermore, the theoretical analysis
proves that the designed algorithm minimizes the processor
usage cost while maintaining the response time limit. For the
request routing, we have developed an RMPR algorithm to
determine the forwarding probability and improve the success
rate of request ﬂows, which is inspired by the clustering and
mapping. Finally, our extensive simulations have shown that the
overall performance of our approach outperforms other bench-
marks. On average, our method improves the request success
rate by 5.47 %when computing resources are sufﬁcient. In addi-
tion, the GMDA-RMPR algorithm not only reduces the average
response latency by 15.1 %in various scenarios, but also cuts the
deployment cost by 27.4 %as compared to other baselines. More-
over, our algorithm minimizes deployment costs while main-
taining low latency and high success rates through probabilistic
mapping of multiple queues and microservice multiplexing. As
further future work, we will consider multi-resource networkenvironments (i.e. bandwidth and database access) as well as di-
verse network topologies and hope to solve the dynamic request
scheduling problem in a real-life microservice architecture.
