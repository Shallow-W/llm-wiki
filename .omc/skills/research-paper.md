# 研究论文 Skill

将一篇学术论文完整摄入 LLM Wiki 知识库的工作流。

## 前置信息收集

在开始前，确认以下信息（如果用户没有提供，主动询问）：

1. **论文来源**：arXiv 链接、本地 PDF 路径、或论文标题
2. **论文标题**（如从链接无法直接推断）
3. **特殊要求**：是否需要特别关注某个方面（如只关注方法、只关注实验等）

## 工作流步骤

### 步骤 1：下载论文

如果论文尚未在 `raw/papers/` 中：

```bash
# 从 arXiv 下载（将 ARXIV_ID 替换为实际 ID）
curl -sLfo "raw/papers/<描述性文件名>.pdf" "https://arxiv.org/pdf/<ARXIV_ID>"
```

文件命名规范：使用英文小写横线连接的描述性名称，如 `evolver-from-trajectories-to-principles.pdf`。

如果用户已提供本地路径，确认文件存在即可。

### 步骤 2：提取 PDF 全文

使用 Python PyMuPDF 提取文本：

```bash
python -c "
import sys, io
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')
import fitz
doc = fitz.open(r'<PDF路径>')
print(f'Pages: {doc.page_count}')
for i in range(doc.page_count):
    page = doc[i]
    print(f'--- Page {i+1} ---')
    print(page.get_text())
" 2>&1
```

注意 UTF-8 编码处理（Windows 上需要 `sys.stdout` 包装）。

### 步骤 3：创建来源摘要页

在 `wiki/sources/<短横线标题>.md` 创建文件，严格遵循以下模板：

```markdown
---
id: source-<短横线标题>
date: YYYY-MM-DD
source: raw/papers/<文件名>.pdf
type: 论文
tags: [相关标签]
---

# 论文完整标题

## 一句话总结
1-2 句话概括核心贡献。

## 基本信息
| 项目 | 内容 |
|------|------|
| **标题** | ... |
| **作者** | ... |
| **机构** | ... |
| **日期** | ... |
| **arXiv** | ... |

## 研究背景与动机
- 解决什么问题？
- 为什么现有方法不够？
- 本文的核心洞察是什么？

## 核心方法
详细描述，包含：
- 关键数学公式（用 LaTeX $...$）
- 架构设计的直觉解释
- 与现有方法的关键区别

**计算流程部分**（精读要求）：
- 提供完整的符号表（每个变量的维度、含义、直觉理解）
- 逐步拆解每个公式（维度变换、直觉类比）
- ASCII 架构图/信号流图
- 具体的数值例子（如维度大小）

## 实验结果
关键数据表格 + 分析。

## 消融实验/分析
（如有）

## 优点与局限
### 优点
### 局限与开放问题

## 与知识库中已有论文的关联
分析与已有页面的联系与区别。

## 提及的实体
- [[实体名]] — 角色

## 讨论的概念
- [[概念名]] — 在此处的用法

## 推荐参考文献
从论文参考文献中挑选 5-10 篇最有趣的，按"强烈推荐/值得阅读/背景补充"分类。

## 关联
- 链接到其他知识库页面
```

### 步骤 4：创建概念页

如果论文提出了重要的新概念/方法/技术，在 `wiki/concepts/<概念名>.md` 创建概念页：

```markdown
---
id: concept-<短横线名>
date: YYYY-MM-DD
aliases: [别名列表]
tags: [概念, 相关标签]
---

# 概念名

## 定义
清晰定义。

## 关键属性
- 属性列表

## 相关概念
- [[相关概念]] — 关系

## 来源
- [[来源标题]] — 概念出现的位置

## 参见
- [[相关页面]]
```

### 步骤 5：创建实体页

为论文的主要作者（一作、通讯作者）和机构创建实体页 `wiki/entities/<实体名>.md`：

```markdown
---
id: entity-<短横线名>
date: YYYY-MM-DD
aliases: []
tags: [实体, 人物/组织/产品]
---

# 实体名

## 概述
1-2 句话说明。

## 关键事实
- 所属机构：...
- 研究方向：...
- 相关论文：[[来源标题]]

## 在来源中的出现
- [[来源标题]] — 角色

## 关系
- [[相关实体]] — 关系描述
```

### 步骤 6：更新索引

编辑 `wiki/index.md`：
1. 更新 `最后更新` 日期
2. 在对应分类（来源/实体/概念）中添加新条目
3. 更新分类计数

### 步骤 7：记录日志

在 `log/YYYY/MM-DD.md` 追加日志条目：

```markdown
## [HH:MM] 摄入 | 论文标题
- 新增：[[来源标题]]
- 新增概念：[[概念A]]、[[概念B]]
- 新增实体：[[实体A]]、[[实体B]]
- 更新索引：来源 N→N+1，概念 M→M+K
- 备注：简短说明核心贡献和与已有知识库的关联
```

## 写作规范

1. **语言**：全部用中文写作
2. **详细程度**：来源摘要要详细（论文的每个关键部分都要覆盖），不要简略
3. **数学公式**：用 LaTeX `$...$` 和 `$$...$$`
4. **交叉引用**：使用 `[[wikilink]]` 链接到已有知识库页面
5. **符号解释**：对计算流程中的每个变量给出维度、含义和直觉理解
6. **参考文献**：从论文的参考文献中挑选最有趣的推荐给用户
7. **关联分析**：分析新论文与知识库中已有论文的联系与区别

## 完成后

向用户报告：
1. 创建了哪些页面
2. 知识库的数量变化（来源/概念/实体）
3. 论文的核心发现摘要
4. 推荐的后续阅读
